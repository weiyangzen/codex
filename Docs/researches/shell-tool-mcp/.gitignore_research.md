# shell-tool-mcp/.gitignore 研究文档

## 场景与职责

`.gitignore` 文件位于 `shell-tool-mcp` 目录下，是 Node.js/TypeScript 项目的标准 Git 忽略配置文件。其核心职责是：

1. **排除构建产物**：防止编译生成的二进制文件被提交到版本控制
2. **排除依赖目录**：防止 `node_modules` 被提交（应通过 package.json 重建）
3. **保持仓库整洁**：只保留源代码和必要配置

## 功能点目的

### 忽略规则说明

| 模式 | 含义 | 目的 |
|------|------|------|
| `/bin/` | 忽略根目录下的 bin 文件夹 | 排除 TypeScript 编译输出（由 tsup.config.ts 配置输出到 bin/） |
| `/node_modules/` | 忽略根目录下的 node_modules | 排除 npm/pnpm 安装的依赖包 |

### 前缀 `/` 的重要性

- 使用 `/` 前缀表示**仅匹配当前目录**下的指定文件夹
- 防止意外忽略子目录中同名文件夹（如 `src/bin/` 不会被此规则影响）
- 这是最佳实践，确保忽略规则的精确性

## 具体技术实现

### 与构建流程的关联

```
构建流程：
1. tsup.config.ts 定义 entry: "src/index.ts" -> outDir: "bin"
2. npm run build 调用 tsup 生成 bin/mcp-server.js
3. .gitignore 确保 bin/ 目录不会被 git 追踪
```

### 与包发布的关联

package.json 中定义了：
```json
"files": ["vendor", "README.md"]
```

注意：
- `bin/` 被 git 忽略
- 但 package.json 的 `files` 字段**不包含** `bin/`
- 这意味着编译后的 JS 不会随 npm 包发布
- 实际发布的是 `vendor/` 目录下的原生二进制文件（Bash/Zsh）

## 关键代码路径与文件引用

| 文件 | 关联关系 |
|------|----------|
| `tsup.config.ts` | 配置输出目录为 `bin/`，与 .gitignore 对应 |
| `package.json` | `scripts.build` 调用 tsup，`files` 字段决定发布内容 |
| `package.json` | `scripts.clean` 执行 `rm -rf bin`，与 .gitignore 配合清理 |

## 依赖与外部交互

- **Git**：Git 工具读取此文件决定哪些文件不追踪
- **npm/pnpm**：与 `packageManager` 字段配合，确保依赖管理一致性
- **tsup**：构建工具，输出到被忽略的目录

## 风险、边界与改进建议

### 当前风险

1. **bin/ 目录双重排除**：
   - 被 .gitignore 排除
   - 也不在 package.json files 中
   - 如果未来需要发布编译后的 JS，需要修改配置

2. **潜在的缓存问题**：
   - 如果开发者手动创建了 `bin/` 并放入重要文件，会被意外忽略

### 改进建议

1. **添加注释说明**：
   ```gitignore
   # Build output (TypeScript compilation)
   /bin/
   
   # Dependencies (recreated by pnpm install)
   /node_modules/
   ```

2. **考虑添加其他常见忽略项**：
   ```gitignore
   # Logs
   *.log
   npm-debug.log*
   
   # IDE
   .idea/
   .vscode/
   
   # Test coverage
   coverage/
   
   # OS files
   .DS_Store
   Thumbs.db
   ```

3. **明确构建产物策略**：
   - 如果 `bin/` 确实不需要发布，当前配置正确
   - 如果需要发布，应将其加入 `package.json` 的 `files` 字段
