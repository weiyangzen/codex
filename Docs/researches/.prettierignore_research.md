# .prettierignore 研究文档

## 场景与职责

`.prettierignore` 是 **Prettier** 代码格式化工具的配置文件，用于指定哪些文件和目录不应该被 Prettier 格式化。该文件位于项目根目录，确保生成的文件、依赖目录、锁文件等不会被 Prettier 修改，避免不必要的格式变更和潜在的问题。

Prettier 是一个固执己见的代码格式化工具，支持多种语言，包括 JavaScript、TypeScript、JSON、Markdown、YAML 等。`.prettierignore` 与 `.gitignore` 功能类似，但专门针对 Prettier 的格式化行为。

## 功能点目的

### 1. 构建输出目录

```
/codex-cli/dist
/codex-cli/node_modules
```

**目的**：排除 codex-cli 包的构建输出和依赖目录。

**技术细节**：
- `/codex-cli/dist`：CLI 包的编译输出目录
- `/codex-cli/node_modules`：CLI 包的本地依赖

**注意**：路径以 `/` 开头，表示相对于工作区根目录。

### 2. 锁文件

```
pnpm-lock.yaml
```

**目的**：排除 pnpm 的锁文件。

**技术背景**：
- 锁文件是机器生成的，有特定的格式要求
- Prettier 可能改变 YAML 格式，导致锁文件无效
- 锁文件应该保持原样，由 pnpm 管理

**相关配置**：
- `.gitignore` 保留了 `!pnpm-lock.yaml`（不忽略）
- `.prettierignore` 排除了 `pnpm-lock.yaml`（不格式化）
- 两者配合：锁文件被版本控制，但不被格式化

### 3. 提示文件

```
prompt.md
*_prompt.md
*_instructions.md
```

**目的**：排除 AI 提示和指令文件。

**使用场景**：
- `prompt.md`：AI 助手的提示文件
- `*_prompt.md`：特定任务的提示文件
- `*_instructions.md`：指令文件

**原因**：
- 这些文件通常包含特定的格式要求
- 格式化可能破坏提示的结构和语义
- 可能包含特殊的标记或占位符

## 具体技术实现

### 文件格式

`.prettierignore` 使用与 `.gitignore` 相同的语法：

```
# 注释
pattern/          # 匹配目录
pattern           # 匹配文件和目录
*.ext             # 通配符
!pattern          # 否定模式（不忽略）
path/pattern      # 特定路径的匹配
```

### 路径解析

```
/codex-cli/dist     # 绝对路径（从工作区根目录）
codex-cli/dist      # 相对路径（任何位置的 codex-cli/dist）
```

当前配置使用绝对路径（以 `/` 开头），确保只忽略根目录下的特定路径。

### 与 .gitignore 的关系

| 文件 | 用途 | 范围 |
|------|------|------|
| `.gitignore` | Git 版本控制 | 所有文件 |
| `.prettierignore` | Prettier 格式化 | 支持的文件类型 |

**重叠内容**：
- 两者都排除 `node_modules`
- `.gitignore` 排除范围更广（包括 `.git`, `bazel-*` 等）

## 关键代码路径与文件引用

### 相关文件

1. **.prettierrc.toml**
   - Prettier 的主要配置文件
   - 定义格式化规则（printWidth, tabWidth 等）
   - `.prettierignore` 与 `.prettierrc.toml` 配合使用

2. **package.json**
   - 包含 Prettier 的 npm 脚本
   ```json
   "scripts": {
     "format": "prettier --check *.json *.md docs/*.md .github/workflows/*.yml **/*.js",
     "format:fix": "prettier --write *.json *.md docs/*.md .github/workflows/*.yml **/*.js"
   }
   ```

3. **pnpm-lock.yaml**
   - 被 `.prettierignore` 排除
   - 由 pnpm 管理格式

4. **codex-cli/package.json**
   - CLI 包的配置
   - 其 `dist` 和 `node_modules` 被排除

### Prettier 工作流程

```
1. Prettier 接收文件路径或 glob 模式
2. 检查 .prettierignore
3. 如果文件匹配忽略模式，跳过格式化
4. 否则，应用 .prettierrc.toml 的规则
5. 输出格式化后的内容
```

## 依赖与外部交互

### Prettier 工具

**安装方式**：
```bash
# npm
npm install --save-dev prettier

# pnpm
pnpm add -D prettier
```

**基本使用**：
```bash
# 检查格式
prettier --check .

# 修复格式
prettier --write .

# 检查特定文件
prettier --check src/**/*.ts

# 使用特定配置
prettier --config .prettierrc.toml --check .
```

### 与 CI/CD 的集成

当前 `package.json` 中的脚本：
```json
"format": "prettier --check *.json *.md docs/*.md .github/workflows/*.yml **/*.js",
"format:fix": "prettier --write *.json *.md docs/*.md .github/workflows/*.yml **/*.js"
```

**注意**：脚本中显式指定了文件模式，`.prettierignore` 仍然生效。

### 编辑器集成

**VS Code**：
- 扩展：esbenp.prettier-vscode
- 自动读取 `.prettierignore`
- 保存时自动格式化（如果启用）

**Vim**：
- 插件：ALE 或 coc-prettier

## 风险、边界与改进建议

### 潜在风险

1. **范围过于狭窄**
   - 当前仅排除了 codex-cli 的特定目录
   - 其他包的 `dist` 和 `node_modules` 可能被格式化

2. **锁文件处理**
   - 仅排除了 `pnpm-lock.yaml`
   - 如果有 `package-lock.json` 或 `yarn.lock`，也需要排除

3. **生成文件**
   - 没有排除所有生成文件（如 OpenAPI 生成的代码）

### 边界情况

1. **符号链接**
   - Prettier 对符号链接的处理可能因版本而异
   - 被忽略目录中的符号链接可能仍被处理

2. **嵌套配置**
   - 子目录可以有独立的 `.prettierignore`
   - 当前项目结构简单，没有嵌套配置

3. **与 ESLint 的冲突**
   - 某些 ESLint 规则可能与 Prettier 冲突
   - 需要 `eslint-config-prettier` 来禁用冲突规则

### 改进建议

1. **扩展忽略范围**
   ```
   # Build outputs
   dist/
   build/
   out/
   
   # Dependencies
   node_modules/
   
   # Lock files
   pnpm-lock.yaml
   package-lock.json
   yarn.lock
   
   # Generated files
   *.generated.ts
   *.gen.ts
   
   # Coverage
   coverage/
   
   # Cache
   .cache/
   ```

2. **添加注释说明**
   ```
   # Prettier ignore patterns
   # See: https://prettier.io/docs/en/ignore.html
   
   # Build outputs
   /codex-cli/dist
   
   # Dependencies
   /codex-cli/node_modules
   
   # Lock files (managed by package manager)
   pnpm-lock.yaml
   
   # AI prompts and instructions
   prompt.md
   *_prompt.md
   *_instructions.md
   ```

3. **与 .gitignore 同步**
   - 考虑使用 `.gitignore` 作为基础
   - 添加 Prettier 特定的排除项

4. **添加测试相关排除**
   ```
   # Test fixtures
   **/__fixtures__/**
   **/test/fixtures/**
   
   # Snapshots
   **/__snapshots__/**
   *.snap
   ```

5. **考虑使用 .prettierignore 的替代方案**
   ```javascript
   // .prettierrc.js
   module.exports = {
     ignorePatterns: [
       "dist/**",
       "node_modules/**",
       "pnpm-lock.yaml"
     ]
   };
   ```

### 使用示例

```bash
# 检查所有文件（遵循 .prettierignore）
npx prettier --check .

# 检查特定文件（即使它在 .prettierignore 中）
npx prettier --check --ignore-path "" codex-cli/dist/file.js

# 查看哪些文件会被格式化
npx prettier --list-different .

# 格式化并输出到 stdout（不写入文件）
npx prettier --stdin-filepath file.ts < file.ts
```

### 配置对比

| 配置项 | .prettierignore | .prettierrc.toml |
|--------|-----------------|------------------|
| 用途 | 排除文件/目录 | 定义格式化规则 |
| 语法 | .gitignore 风格 | TOML/JSON/JS |
| 示例 | `node_modules/` | `printWidth = 80` |

**最佳实践**：
- 使用 `.prettierignore` 排除不应该格式化的文件
- 使用 `.prettierrc.toml` 定义格式规则
- 保持两者同步，避免意外格式化
