# .bazelrc 研究文档

## 场景与职责

`.bazelrc` 是 Bazel 构建系统的配置文件，位于项目根目录，用于定义默认的构建选项、环境变量和工具链配置。该文件确保所有开发者和 CI 环境使用一致的 Bazel 构建设置，实现可重现的构建。

Codex 项目使用 Bazel 作为主要的构建系统（与 Cargo 并行），`.bazelrc` 配置了跨平台支持、远程缓存、远程执行等高级功能。

## 功能点目的

### 1. 工具链配置

```
common --repo_env=BAZEL_DO_NOT_DETECT_CPP_TOOLCHAIN=1
common --repo_env=BAZEL_NO_APPLE_CPP_TOOLCHAIN=1
common --xcode_version_config=//:disable_xcode
```

**目的**：禁用自动工具链检测，使用项目定义的 LLVM 工具链。

**技术细节**：
- `BAZEL_DO_NOT_DETECT_CPP_TOOLCHAIN=1`：阻止 Bazel 自动检测系统 C++ 编译器
- `BAZEL_NO_APPLE_CPP_TOOLCHAIN=1`：在 macOS 上不使用 Xcode 工具链
- `disable_xcode`：使用自定义的 Xcode 配置占位符，避免构建 xcode_locator

### 2. 缓存配置

```
common --disk_cache=~/.cache/bazel-disk-cache
common --repo_contents_cache=~/.cache/bazel-repo-contents-cache
common --repository_cache=~/.cache/bazel-repo-cache
common --remote_cache_compression
startup --experimental_remote_repo_contents_cache
```

**目的**：配置多级缓存系统以加速构建。

**缓存层级**：
1. **Disk Cache** (`~/.cache/bazel-disk-cache`)：本地磁盘缓存构建产物
2. **Repo Contents Cache** (`~/.cache/bazel-repo-contents-cache`)：仓库内容缓存
3. **Repository Cache** (`~/.cache/bazel-repo-cache`)：外部依赖缓存
4. **Remote Cache Compression**：启用远程缓存压缩减少传输

### 3. 平台配置

```
common --experimental_platform_in_output_dir
common --enable_platform_specific_config
common:linux --host_platform=//:local_linux
common:windows --host_platform=//:local_windows
```

**目的**：启用平台特定配置，支持跨平台构建。

**平台定义**（见 `BUILD.bazel`）：
- `local_linux`：标记为 glibc 2.28 兼容
- `local_windows`：使用 gnullvm ABI

### 4. Runfiles 配置

```
common --noenable_runfiles
```

**目的**：禁用 runfiles 树，使用替代机制。

**背景**：见 `codex-rs/utils/cargo-bin/README.md`，该目录包含关于 runfiles 策略的详细说明。

### 5. 远程构建执行 (RBE) 配置

```
common --test_output=errors
common --bes_results_url=https://app.buildbuddy.io/invocation/
common --bes_backend=grpcs://remote.buildbuddy.io
common --remote_cache=grpcs://remote.buildbuddy.io
common --remote_download_toplevel
common --nobuild_runfile_links
common --remote_timeout=3600
common --noexperimental_throttle_remote_action_building
common --experimental_remote_execution_keepalive
common --grpc_keepalive_time=30s
common --jobs=30
```

**目的**：配置 BuildBuddy 远程构建服务。

**关键配置**：
- **BES (Build Event Stream)**：构建事件流后端
- **Remote Cache**：远程缓存端点
- **Remote Execution**：远程执行（通过 `--config=remote` 启用）
- **Jobs**：并发任务数限制为 30

### 6. 远程执行专用配置

```
common:remote --extra_execution_platforms=//:rbe
common:remote --remote_executor=grpcs://remote.buildbuddy.io
common:remote --jobs=800
```

**目的**：启用远程执行模式，使用 800 个并发任务。

**使用方式**：
```bash
bazel build --config=remote //...
```

### 7. 环境变量传递

```
common --incompatible_strict_action_env
common:linux --test_env=PATH=/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin
common:macos --test_env=PATH=/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin
common:windows --test_env=PATH
common:windows --test_env=SYSTEMROOT
common:windows --test_env=COMSPEC
common:windows --test_env=WINDIR
```

**目的**：
- 启用严格的 action 环境（`incompatible_strict_action_env`）
- 为不同平台配置测试时的 PATH 环境变量
- Windows 需要额外的系统环境变量

## 具体技术实现

### 配置语法

```
<command>:<config> --<option>=<value>
```

- `common`：适用于所有命令
- `common:linux`：仅适用于 Linux 平台
- `common:macos`：仅适用于 macOS 平台  
- `common:windows`：仅适用于 Windows 平台
- `common:remote`：仅在指定 `--config=remote` 时生效
- `startup`：Bazel 启动选项

### 平台检测

Bazel 通过 `--enable_platform_specific_config` 自动检测主机平台并应用相应配置。

### 与 MODULE.bazel 的协作

```
# MODULE.bazel 定义依赖和工具链
# .bazelrc 配置如何使用这些工具链
```

## 关键代码路径与文件引用

### 相关文件

1. **MODULE.bazel**
   - 定义 Bazel 模块和外部依赖
   - 配置 Rust 工具链版本（1.93.0）
   - 注册 LLVM 工具链

2. **BUILD.bazel** (根目录)
   - 定义平台配置 (`local_linux`, `local_windows`)
   - 定义 `rbe` 平台别名
   - 包含 Xcode 禁用配置

3. **rbe.bzl**
   - 定义远程执行平台仓库规则
   - 配置容器镜像和平台约束

4. **.github/workflows/bazel.yml**
   - CI 工作流使用 `.bazelrc` 配置
   - 根据 API key 可用性切换本地/远程模式

### 构建流程

```
1. Bazel 启动，读取 .bazelrc
2. 应用 startup 选项
3. 检测平台，应用平台特定配置
4. 解析命令行，可能应用 --config=remote
5. 配置缓存和远程执行后端
6. 执行构建/测试
```

## 依赖与外部交互

### 外部服务

1. **BuildBuddy** (https://buildbuddy.io)
   - 远程缓存和远程执行服务
   - 需要 `BUILDBUDDY_API_KEY` 环境变量
   - 支持 BES (Build Event Stream)

2. **Docker Hub**
   - 远程执行使用容器镜像：`mbolin491/codex-bazel`
   - 镜像包含 git、python3、dotslash 等工具

### 工具链依赖

```
LLVM Toolchain (@llvm//toolchain:all)
  ├── clang 编译器
  ├── libc (glibc 2.28)
  └── 平台约束定义

Rust Toolchain (@default_rust_toolchains//:all)
  ├── rustc 1.93.0
  ├── cargo
  └── 2024 edition
```

### 与 Cargo 的交互

- `crate.from_cargo()` 在 `MODULE.bazel` 中从 `Cargo.lock` 导入依赖
- `.bazelrc` 配置的缓存独立于 Cargo 的 `target` 目录
- 两者可以并行使用，但缓存不共享

## 风险、边界与改进建议

### 潜在风险

1. **缓存不一致**
   - 远程缓存可能包含过期的构建产物
   - 建议定期清理或使用 `--nocache_test_results`

2. **平台特定问题**
   - Windows 配置相对简单，可能需要更多测试
   - ARM64 构建在 CI 中被注释掉（见 bazel.yml）

3. **远程执行依赖**
   - 需要有效的 BuildBuddy API key
   - 社区贡献者可能无法使用远程执行

4. **环境变量泄漏**
   - `incompatible_strict_action_env` 减少了环境变量泄漏
   - 但某些工具可能依赖未传递的环境变量

### 边界情况

1. **离线构建**
   - 没有网络连接时，远程缓存和执行不可用
   - 本地磁盘缓存仍然有效

2. **多用户环境**
   - 缓存目录使用用户主目录，多用户系统可能有权限问题

3. **CI 环境**
   - GitHub Actions 的缓存策略需要与 `.bazelrc` 协调
   - 当前 CI 配置禁用了某些缓存（见 bazel.yml 注释）

### 改进建议

1. **添加配置文档**
   ```bash
   # 在 .bazelrc 顶部添加
   # 配置使用说明：
   # - 本地开发：bazel build //...
   # - 远程缓存：bazel build //... (需要 BUILDBUDDY_API_KEY)
   # - 远程执行：bazel build --config=remote //...
   ```

2. **优化 Windows 支持**
   - 添加更多 Windows 特定的测试环境变量
   - 考虑 PowerShell 执行策略

3. **缓存策略优化**
   ```
   # 考虑添加本地开发优化
   common:local --disk_cache=~/.cache/bazel-disk-cache-dev
   common:local --repository_cache=~/.cache/bazel-repo-cache-dev
   ```

4. **调试支持**
   ```
   # 添加调试配置
   common:debug --verbose_failures
   common:debug --sandbox_debug
   common:debug --explain=explain.log
   ```

5. **性能监控**
   ```
   # 添加性能分析配置
   common:profile --profile=profile.gz
   common:profile --slim_profile
   ```

6. **定期审查**
   - 随着 Bazel 版本升级，某些实验性选项可能变为默认
   - 建议跟踪 Bazel 发布说明，更新配置

### 配置使用示例

```bash
# 本地开发构建
bazel build //codex-rs/cli:codex

# 使用远程缓存（如果有 API key）
bazel test //...

# 使用远程执行
bazel build --config=remote //...

# 特定平台构建
bazel build --config=linux //...
bazel build --config=macos //...
bazel build --config=windows //...

# 清理缓存后构建
bazel clean --expunge
bazel build //...
```
