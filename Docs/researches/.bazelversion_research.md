# .bazelversion 研究文档

## 场景与职责

`.bazelversion` 文件是 Bazel 版本管理工具 **Bazelisk** 的配置文件，用于指定项目所需的 Bazel 版本。Bazelisk 是 Bazel 的推荐启动器，类似于 Node.js 的 nvm 或 Python 的 pyenv，确保所有开发者和 CI 环境使用完全相同的 Bazel 版本。

该文件位于项目根目录，内容简单直接：仅包含版本号字符串。

## 功能点目的

### 1. 版本锁定

```
9.0.0
```

**目的**：锁定 Bazel 版本为 9.0.0。

**技术意义**：
- 确保构建的可重现性
- 避免不同 Bazel 版本之间的行为差异
- 防止新版本的破坏性变更影响项目

### 2. Bazelisk 集成

**目的**：与 Bazelisk 工具配合，自动下载和使用指定版本。

**工作流程**：
1. 开发者运行 `bazel` 命令（实际调用 Bazelisk）
2. Bazelisk 读取 `.bazelversion` 文件
3. 如果指定版本未安装，自动下载
4. 启动对应版本的 Bazel 二进制文件

## 具体技术实现

### 文件格式

`.bazelversion` 是一个纯文本文件，包含单行版本号：

```
<major>.<minor>.<patch>
```

支持格式：
- 精确版本：`9.0.0`
- 最新版本：`latest`
- 特定发布：`latest-1`（倒数第二个版本）
- Git commit hash（用于开发版本）

### 版本解析流程

```
1. Bazelisk 启动
2. 从当前目录向上查找 .bazelversion
3. 读取版本字符串
4. 检查 ~/.bazelisk/cache 是否已有该版本
5. 如无，从 GitHub releases 下载
6. 启动对应版本的 bazel 二进制文件
```

## 关键代码路径与文件引用

### 相关文件

1. **MODULE.bazel**
   - 使用 Bazel 9.0 的新特性
   - 模块系统（bzlmod）在 Bazel 9 中更加成熟

2. **MODULE.bazel.lock**
   - 模块依赖锁定文件
   - 与 Bazel 9.0 的锁定格式兼容

3. **.bazelrc**
   - 配置选项针对 Bazel 9.0 优化
   - 使用 `--experimental_remote_repo_contents_cache` 等实验性特性

4. **.github/workflows/bazel.yml**
   - CI 使用 `bazelbuild/setup-bazelisk@v3` 动作
   - 自动读取 `.bazelversion`

### 版本兼容性

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| Bazel | 9.0.0 | 核心构建系统 |
| rules_rust | 0.0.43 | Rust 规则集 |
| bazel_dep | MODULE.bazel | 模块依赖系统 |

## 依赖与外部交互

### Bazelisk 工具

**安装方式**：
```bash
# macOS
brew install bazelisk

# Linux
npm install -g @bazel/bazelisk

# 或下载二进制
wget https://github.com/bazelbuild/bazelisk/releases/latest/download/bazelisk-linux-amd64
```

**环境变量**：
- `BAZELISK_HOME`：缓存目录（默认 `~/.bazelisk`）
- `USE_BAZEL_VERSION`：覆盖 `.bazelversion` 的值

### Bazel 9.0 新特性

**主要改进**（与项目相关）：
1. **bzlmod 默认启用**：模块系统更加稳定
2. **改进的远程执行**：更好的远程缓存支持
3. **平台工具链**：更灵活的平台和工具链配置
4. **性能优化**：更快的增量构建

**破坏性变更**（需要关注）：
- 某些旧版规则可能需要更新
- 命令行选项可能有变化

### 与 CI/CD 的集成

```yaml
# .github/workflows/bazel.yml
- name: Set up Bazel
  uses: bazelbuild/setup-bazelisk@v3
```

`setup-bazelisk` 动作自动：
1. 安装 Bazelisk
2. 读取 `.bazelversion`
3. 预下载指定版本的 Bazel

## 风险、边界与改进建议

### 潜在风险

1. **版本过旧**
   - Bazel 9.0.0 发布于 2024 年底
   - 需要关注安全更新和 bug 修复
   - 建议定期评估升级

2. **下载依赖**
   - 新开发者首次运行时需要下载 Bazel
   - 网络问题可能导致下载失败
   - CI 环境需要稳定的网络连接

3. **平台支持**
   - Bazel 9.0 可能不支持某些旧平台
   - 需要验证所有目标平台的兼容性

### 边界情况

1. **无 Bazelisk 环境**
   - 如果系统直接安装 `bazel` 而非 `bazelisk`
   - `.bazelversion` 被忽略，使用系统默认版本
   - 可能导致版本不匹配

2. **版本字符串格式**
   - 多余的空格或换行可能导致解析失败
   - 建议使用 `echo -n "9.0.0" > .bazelversion` 避免换行

3. **子目录版本**
   - Bazelisk 从当前目录向上查找
   - 子目录可以有独立的 `.bazelversion`

### 改进建议

1. **添加版本说明**
   ```bash
   # 在 .bazelversion 同级创建 .bazelversion.md
   # Bazel 9.0.0 选择理由：
   # - bzlmod 模块系统稳定
   # - 改进的远程执行支持
   # - 与 rules_rust 0.0.43 兼容
   ```

2. **版本升级流程**
   ```bash
   # 创建升级检查清单
   # 1. 检查 Bazel 发布说明
   # 2. 在本地测试构建
   # 3. 更新 .bazelversion
   # 4. 运行完整 CI 测试
   # 5. 更新文档
   ```

3. **备用版本支持**
   ```bash
   # 创建 .bazelversion 的备选机制
   # 如支持 .bazeliskrc 文件
   ```

4. **定期更新策略**
   - 每季度评估 Bazel 新版本
   - 关注 LTS 版本发布
   - 建立升级测试流程

bazel --print_env 2>&1 | head

# 清理 Bazelisk 缓存
