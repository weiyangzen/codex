# .codespellignore 研究文档

## 场景与职责

`.codespellignore` 文件是 **codespell** 工具的自定义忽略词列表。codespell 是一个用于检查源代码中常见拼写错误的工具，它通过查找已知的拼写错误模式来工作。然而，某些在代码中合法出现的词汇（如专有名词、缩写、技术术语）可能被误判为拼写错误，因此需要显式声明忽略。

该文件与 `.codespellrc` 配合使用，后者定义了 codespell 的整体行为，而 `.codespellignore` 专门列出需要忽略的词汇。

## 功能点目的

### 1. 终端模拟器名称

```
iTerm
iTerm2
```

**目的**：忽略 iTerm 和 iTerm2（macOS 流行的终端模拟器）的名称。

**技术背景**：
- iTerm 是 macOS 上最流行的终端模拟器之一
- "iTerm" 可能被误判为 "item" 的拼写错误
- 在 Codex 项目中，TUI（终端用户界面）组件可能需要检测或提及 iTerm

### 2. 常见缩写和术语

```
psuedo
te
TE
```

**目的**：忽略特定的缩写和技术术语。

**详细说明**：
- **psuedo**：可能是 "pseudo"（伪）的常见拼写变体，在某些上下文中故意使用
- **te**：可能是 "text editor" 或其他技术术语的缩写
- **TE**：可能是 "Terminal Emulator" 或其他大写缩写的简称

**注意**：`psuedo` 实际上是一个常见的拼写错误（正确应为 `pseudo`），但可能在某些历史代码或特定上下文中被保留。

## 具体技术实现

### 文件格式

`.codespellignore` 是简单的换行分隔文本文件：
- 每行一个要忽略的单词
- 大小写敏感（`te` 和 `TE` 分别列出）
- 不支持通配符或正则表达式
- 空行通常被忽略

### 与 .codespellrc 的关系

```
.codespellrc (主配置)
  ├── skip = ... (跳过文件模式)
  ├── ignore-regex = ... (正则忽略)
  └── ignore-words-list = ... (内联忽略词列表)

.codespellignore (扩展忽略词列表)
  └── 每行一个忽略词
```

在 `.codespellrc` 中：
```
ignore-words-list = ratatui,ser,iTerm,iterm2,iterm,te,TE
```

注意：`.codespellignore` 文件的内容与 `ignore-words-list` 有重叠，但 `.codespellignore` 提供了更清晰的维护方式。

### Codespell 工作流程

```
1. Codespell 扫描源文件
2. 匹配已知拼写错误字典
3. 检查 .codespellignore 和 ignore-words-list
4. 如果匹配忽略列表，跳过该词
5. 输出剩余的潜在拼写错误
```

## 关键代码路径与文件引用

### 相关文件

1. **.codespellrc**（主配置文件）
   - 定义扫描范围、跳过模式、正则忽略
   - 包含内联的 `ignore-words-list`
   - 引用 `.codespellignore` 作为额外忽略词源

2. **.github/workflows/codespell.yml**（CI 工作流）
   ```yaml
   - name: Codespell
     uses: codespell-project/actions-codespell@v2
     with:
       ignore_words_file: .codespellignore
   ```

3. **AGENTS.md**
   - 项目文档中可能包含这些术语
   - 确保文档中的专有名词不被标记

### 扫描范围

根据 `.codespellrc`：
```
skip = .git*,vendor,*-lock.yaml,*.lock,.codespellrc,*test.ts,*.jsonl,frame*.txt,*.snap,*.snap.new,*meriyah.umd.min.js
```

扫描包括：
- 源代码文件
- 文档文件（Markdown）
- 配置文件
- 排除：生成的文件、锁文件、测试快照等

## 依赖与外部交互

### Codespell 工具

**安装方式**：
```bash
# pip
pip install codespell

# Homebrew
brew install codespell

# GitHub Actions
codespell-project/actions-codespell@v2
```

**字典来源**：
- codespell 内置常见拼写错误字典
- 可以通过 `--dictionary` 添加自定义字典
- `.codespellignore` 作为负向字典（排除项）

### CI/CD 集成

```yaml
# .github/workflows/codespell.yml
name: Codespell
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  codespell:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
      - uses: codespell-project/actions-codespell@v2
        with:
          ignore_words_file: .codespellignore
```

### 与编辑器集成

许多编辑器支持 codespell 集成：
- VS Code: Code Spell Checker 扩展
- Vim: ALE 或 coc-codespell
- Emacs: flyspell 或 codespell 插件

## 风险、边界与改进建议

### 潜在风险

1. **过度忽略**
   - 将真正的拼写错误加入忽略列表
   - 建议：定期审查忽略列表，移除不再需要的项

2. **大小写敏感问题**
   - `te` 和 `TE` 分别列出，可能遗漏 `Te` 等变体
   - 建议：考虑添加常见大小写变体

3. **词汇冲突**
   - 忽略的词汇可能在其他上下文中确实是错误
   - 例如：`psuedo` 在大多数情况下是 `pseudo` 的拼写错误

### 边界情况

1. **部分匹配**
   - codespell 可能匹配单词的一部分
   - 例如：`iTerm` 可能匹配 `term` 的变体

2. **多语言支持**
   - 当前配置主要针对英语
   - 代码中的非英语注释可能被误判

3. **生成的代码**
   - 某些生成的代码可能包含"错误"的拼写
   - 已通过 `skip` 模式排除大部分生成文件

### 改进建议

1. **添加注释说明**
   ```
   # Terminal emulator names
   iTerm
   iTerm2
   
   # Technical abbreviations
   te
   TE
   
   # Legacy/historical spellings (consider fixing)
   psuedo
   ```

2. **分类组织**
   ```
   # === Terminal Emulators ===
   iTerm
   iTerm2
   
   # === UI Framework ===
   ratatui
   
   # === Abbreviations ===
   te
   TE
   ser  # serde abbreviation
   ```

3. **定期审查流程**
   ```bash
   # 每季度运行一次
   codespell --interactive=3
   # 审查每个"错误"，决定是修复还是添加到忽略列表
   ```

4. **与 .codespellrc 合并**
   - 考虑将所有忽略词统一到一个文件
   - 或者完全分离：`.codespellrc` 用于配置，`.codespellignore` 用于词汇

5. **考虑移除 psuedo**
   ```bash
   # 检查项目中是否真的需要保留 psuedo
   grep -r "psuedo" --include="*.rs" --include="*.ts" --include="*.md" .
   # 如果没有或很少，考虑修复而不是忽略
   ```

### 使用示例

```bash
# 本地运行 codespell
codespell

# 查看将被忽略的词
codespell --debug 2>&1 | grep -i "ignore"

# 交互式修复
codespell --interactive=3

# 仅检查特定目录
codespell codex-rs/src docs/
```

### 相关配置对比

```
# .codespellignore (本文件)
iTerm
iTerm2
psuedo
te
TE

# .codespellrc 中的 ignore-words-list
ratatui,ser,iTerm,iterm2,iterm,te,TE
```

**建议统一策略**：
- 将所有忽略词移到 `.codespellignore`，每行一个（更清晰）
- 或完全使用 `.codespellrc` 的 `ignore-words-list`（更紧凑）
- 当前混合方式可能导致维护困难
