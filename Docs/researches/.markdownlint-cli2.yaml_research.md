# .markdownlint-cli2.yaml 研究文档

## 场景与职责

`.markdownlint-cli2.yaml` 是 **markdownlint-cli2** 工具的配置文件，用于定义 Markdown 文档的 linting 规则和检查范围。markdownlint 是一个用于检查 Markdown 文件风格和格式问题的工具，帮助维护文档的一致性和可读性。

该文件位于项目根目录，专门针对 Codex 项目的文档需求进行配置，特别是针对行长度限制和特定文件的检查范围。

## 功能点目的

### 1. 行长度规则配置

```yaml
config:
  MD013:
    line_length: 100
```

**目的**：配置 Markdown 文件的行长度限制。

**技术细节**：
- **规则 ID**：`MD013`（Line length）
- **限制值**：100 个字符
- **默认行为**：markdownlint 默认限制为 80 字符

**规则说明**：
- `MD013` 规则检查每行是否超过指定长度
- 超过限制的行会被标记为警告或错误
- 有助于提高可读性，特别是在终端和代码编辑器中

**为什么设置为 100**：
- 现代显示器宽度可以容纳更多字符
- 与项目的其他配置保持一致（如 `.prettierrc.toml` 中的 `printWidth = 80`）
- 平衡可读性和内容密度

### 2. 文件范围配置

```yaml
globs:
  - "docs/tui-chat-composer.md"
```

**目的**：指定 markdownlint 应该检查的文件。

**当前配置分析**：
- 仅检查 `docs/tui-chat-composer.md` 一个文件
- 这是一个非常有限的范围

**可能的原因**：
1. **渐进式采用**：项目可能正在逐步引入 markdownlint
2. **特定文档要求**：该文件可能有特殊的格式要求
3. **实验性配置**：可能正在测试 markdownlint 的集成

## 具体技术实现

### 文件格式

`.markdownlint-cli2.yaml` 使用 YAML 格式：

```yaml
# 规则配置
config:
  RULE_ID:
    option: value

# 文件范围
globs:
  - "pattern1"
  - "pattern2"

# 其他选项
fix: true  # 自动修复
```

### markdownlint 规则

**常用规则**（本文件未配置，使用默认）：

| 规则 ID | 描述 | 默认状态 |
|---------|------|---------|
| MD001 | 标题层级递增 | 启用 |
| MD003 | 标题样式一致 | 启用 |
| MD009 | 行尾空格 | 启用 |
| MD010 | 硬制表符 | 启用 |
| MD012 | 连续空行 | 启用 |
| MD013 | 行长度 | 启用（已配置为 100） |
| MD014 | 命令输出前的美元符号 | 启用 |
| MD022 | 标题前后空行 | 启用 |
| MD024 | 重复标题 | 启用 |
| MD025 | 单个一级标题 | 启用 |
| MD031 | 代码块周围空行 | 启用 |
| MD032 | 列表周围空行 | 启用 |
| MD033 | 内联 HTML | 启用 |
| MD034 | 裸 URL | 启用 |
| MD040 | 代码块语言标识 | 启用 |

### 配置优先级

```
1. 命令行参数（最高优先级）
2. 项目根目录的 .markdownlint-cli2.yaml
3. 上级目录的配置文件
4. 用户主目录的配置
5. 默认规则（最低优先级）
```

## 关键代码路径与文件引用

### 相关文件

1. **docs/tui-chat-composer.md**
   - 当前唯一被检查的 Markdown 文件
   - TUI Chat Composer 的文档

2. **docs/** 目录
   - 项目文档目录
   - 包含多个 Markdown 文件

3. **AGENTS.md**
   - 项目的主要文档文件
   - 当前未被 markdownlint 检查

4. **README.md**
   - 项目根目录的说明文件
   - 当前未被 markdownlint 检查

5. **.prettierrc.toml**
   - Prettier 格式化配置
   - `proseWrap = "preserve"` 影响 Markdown 换行

### 与 Prettier 的关系

```toml
# .prettierrc.toml
printWidth = 80
proseWrap = "preserve"
```

| 工具 | 行长度 | 行为 |
|------|--------|------|
| Prettier | 80 | 格式化时可能换行 |
| markdownlint | 100 | 仅检查，不修改 |

**注意**：Prettier 的 `proseWrap = "preserve"` 表示不自动换行 Markdown 文本，这与 markdownlint 的 `MD013` 规则需要协调。

## 依赖与外部交互

### markdownlint-cli2 工具

**安装方式**：
```bash
# npm
npm install -g markdownlint-cli2

# 或作为项目依赖
npm install --save-dev markdownlint-cli2
```

**基本使用**：
```bash
# 检查当前目录的所有 Markdown 文件
markdownlint-cli2 "**/*.md"

# 检查特定文件
markdownlint-cli2 docs/tui-chat-composer.md

# 自动修复问题
markdownlint-cli2 --fix "**/*.md"

# 使用特定配置
markdownlint-cli2 --config .markdownlint-cli2.yaml "**/*.md"
```

### 与 CI/CD 的集成

当前项目似乎没有将 markdownlint 集成到 CI 中。建议添加：

```yaml
# .github/workflows/markdown-lint.yml
name: Markdown Lint
on: [push, pull_request]
jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: DavidAnson/markdownlint-cli2-action@v16
        with:
          globs: "**/*.md"
```

### 编辑器集成

**VS Code**：
- 扩展：DavidAnson.vscode-markdownlint
- 自动读取 `.markdownlint-cli2.yaml`

**Vim**：
- 插件：ALE 或 coc-markdownlint

## 风险、边界与改进建议

### 潜在风险

1. **范围过于有限**
   - 当前仅检查一个文件
   - 其他文档可能存在格式问题

2. **与 Prettier 冲突**
   - Prettier 可能格式化到 80 字符
   - markdownlint 允许到 100 字符
   - 可能导致不一致

3. **缺少 CI 集成**
   - 文档格式问题可能在 PR 中引入
   - 没有自动化检查

### 边界情况

1. **表格和代码块**
   - `MD013` 默认会跳过表格和代码块
   - 但长行在这些元素中仍然可能影响可读性

2. **中文/日文内容**
   - 100 字符限制对 CJK 文字可能过于宽松
   - 每个字符视觉上占用更多空间

3. **链接引用**
   - 长 URL 可能超过行长度限制
   - 需要使用引用式链接

### 改进建议

1. **扩展检查范围**
   ```yaml
   globs:
     - "**/*.md"
     - "!node_modules/**"
     - "!codex-rs/target/**"
   ```

2. **添加更多规则配置**
   ```yaml
   config:
     MD013:
       line_length: 100
       code_blocks: false  # 不检查代码块
       tables: false       # 不检查表格
     MD024:
       siblings_only: true  # 允许不同父标题下的重复标题
     MD033:
       allowed_elements: ["br", "details", "summary"]  # 允许特定 HTML 标签
   ```

3. **与 Prettier 协调**
   ```toml
   # .prettierrc.toml
   printWidth = 100  # 与 markdownlint 一致
   proseWrap = "always"  # 自动换行
   ```
   或
   ```yaml
   # .markdownlint-cli2.yaml
   config:
     MD013: false  # 禁用行长度检查，依赖 Prettier
   ```

4. **添加 CI 集成**
   ```yaml
   # .github/workflows/docs.yml
   name: Documentation
   on:
     push:
       paths:
         - "**/*.md"
     pull_request:
       paths:
         - "**/*.md"
   
   jobs:
     lint:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - uses: DavidAnson/markdownlint-cli2-action@v16
   ```

5. **添加忽略模式**
   ```yaml
   ignores:
     - "**/node_modules/**"
     - "codex-rs/target/**"
     - "CHANGELOG.md"  # 自动生成的变更日志
   ```

6. **启用自动修复**
   ```yaml
   fix: true  # 允许自动修复可修复的问题
   ```

### 使用示例

```bash
# 检查所有 Markdown 文件
npx markdownlint-cli2 "**/*.md"

# 检查并修复
npx markdownlint-cli2 --fix "**/*.md"

# 检查特定规则
npx markdownlint-cli2 --rules MD013,MD024 "**/*.md"

# 生成配置模板
npx markdownlint-cli2 --print-config
```
