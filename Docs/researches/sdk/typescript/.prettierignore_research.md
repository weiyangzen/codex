# .prettierignore 研究文档

## 场景与职责

`.prettierignore` 文件是 Prettier 代码格式化工具的配置文件，用于指定哪些文件或目录应该被 Prettier 忽略而不进行格式化。该文件位于 `sdk/typescript/` 目录下，专门服务于 TypeScript SDK 子项目。

## 功能点目的

该文件的核心目的是排除以下三类目录，避免 Prettier 对这些目录中的文件进行不必要的格式化操作：

1. **`dist`** - 编译输出目录，包含构建后的 JavaScript 文件和类型定义文件
2. **`node_modules`** - 依赖包目录，包含第三方 npm 包
3. **`coverage`** - 测试覆盖率报告目录，由 Jest 生成

## 具体技术实现

### 文件格式

采用简单的行分隔文本格式，每行一个模式：

```
dist
node_modules
coverage
```

### 忽略规则说明

| 目录 | 原因 |
|------|------|
| `dist` | 该目录由 `tsup` 构建工具生成，包含编译后的代码。格式化这些文件没有意义，因为它们会在每次构建时被覆盖 |
| `node_modules` | 第三方依赖包，不应该修改其内部代码格式 |
| `coverage` | Jest 生成的测试覆盖率报告文件，是自动生成的 HTML/JSON 报告 |

### 与 package.json 的关联

在 `package.json` 中定义了以下与 Prettier 相关的脚本：

```json
"format": "prettier --check .",
"format:fix": "prettier --write ."
```

这两个命令都会读取 `.prettierignore` 文件来排除指定目录。

## 关键代码路径与文件引用

- **配置文件位置**: `sdk/typescript/.prettierignore`
- **相关配置文件**: `sdk/typescript/.prettierrc` (格式化规则配置)
- **调用方**: `package.json` 中的 `format` 和 `format:fix` 脚本
- **构建输出目录**: `dist/` (由 `tsup.config.ts` 配置)

## 依赖与外部交互

### 工具依赖

- **Prettier**: 代码格式化工具，版本 `^3.6.2` (定义在 `package.json` 的 `devDependencies` 中)

### 相关配置链

```
package.json scripts -> prettier --check/--write . -> .prettierignore (排除规则)
                                              -> .prettierrc (格式化规则)
```

## 风险、边界与改进建议

### 潜在风险

1. **构建产物被格式化**: 如果 `dist` 目录未被正确忽略，Prettier 可能会格式化构建输出，导致不必要的文件修改和潜在的源映射问题
2. **性能问题**: 如果 `node_modules` 未被忽略，Prettier 会遍历大量第三方文件，严重影响格式化性能

### 边界情况

- 该文件仅影响 Prettier 工具，不影响 ESLint 或其他工具
- 模式匹配采用简单的字符串匹配，不支持通配符或正则表达式
- 文件路径是相对于 `.prettierignore` 所在目录的

### 改进建议

1. **添加注释说明**: 可以为每个忽略项添加注释，说明忽略原因：
   ```
   # Build output
   dist
   
   # Dependencies
   node_modules
   
   # Test coverage reports
   coverage
   ```

2. **考虑添加其他常见忽略项**:
   - `*.log` - 日志文件
   - `.env*` - 环境变量文件
   - `*.min.js` - 压缩后的文件

3. **与 `.gitignore` 同步**: 建议保持与 `.gitignore` 的一致性，避免对不会被提交的文件进行格式化
