# .prettierrc 研究文档

## 场景与职责

`.prettierrc` 是 Prettier 代码格式化工具的配置文件，位于 `sdk/typescript/` 目录下，用于定义 TypeScript SDK 项目的代码格式化规则。该配置文件确保项目中的所有代码遵循统一的格式风格。

## 功能点目的

该配置文件定义了以下核心格式化规则：

1. **控制行宽度** (`printWidth`): 限制每行代码的最大字符数
2. **控制引号风格** (`singleQuote`): 统一使用双引号
3. **控制尾随逗号** (`trailingComma`): 在多行结构中统一使用尾随逗号

## 具体技术实现

### 配置内容

```json
{
  "printWidth": 100,
  "singleQuote": false,
  "trailingComma": "all"
}
```

### 配置项详解

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `printWidth` | `100` | 每行最大字符数为 100，超过则自动换行。默认值是 80，此处放宽以容纳更长的表达式 |
| `singleQuote` | `false` | 使用双引号而非单引号。这与传统 JavaScript 风格一致，也便于在字符串中使用英文缩写 |
| `trailingComma` | `"all"` | 在所有可能的地方使用尾随逗号（包括函数参数）。这有助于减少 Git diff，使代码审查更清晰 |

### 与 ESLint 的协同

项目同时使用 ESLint (`eslint.config.js`) 和 Prettier。为了避免规则冲突：

- `package.json` 中安装了 `eslint-config-prettier` 作为 devDependency
- 该包会禁用 ESLint 中与 Prettier 冲突的格式化规则

## 关键代码路径与文件引用

- **配置文件位置**: `sdk/typescript/.prettierrc`
- **忽略配置**: `sdk/typescript/.prettierignore`
- **调用方**: `package.json` 中的脚本
  ```json
  "format": "prettier --check .",
  "format:fix": "prettier --write ."
  ```
- **ESLint 集成**: `eslint.config.js` (通过 `eslint-config-prettier`)

## 依赖与外部交互

### 工具依赖

- **Prettier**: `^3.6.2` (定义在 `package.json` 的 `devDependencies` 中)
- **eslint-config-prettier**: `^9.1.2` (禁用 ESLint 的格式化规则，避免冲突)

### 配置加载机制

Prettier 会按照以下顺序查找配置：
1. 命令行参数
2. `.prettierrc` 文件
3. `package.json` 中的 `prettier` 字段

本项目采用独立的 `.prettierrc` 文件，便于维护和版本控制。

## 风险、边界与改进建议

### 潜在风险

1. **团队适应性**: `printWidth: 100` 比默认值 80 更宽，可能在某些显示器或分屏编辑时导致水平滚动
2. **与现有代码风格冲突**: 如果项目已有大量代码使用单引号，切换配置会导致大量格式变更

### 边界情况

- Prettier 不会格式化被 `.prettierignore` 忽略的文件
- 某些代码结构（如长字符串模板）可能无法完美适配 `printWidth`
- `trailingComma: "all"` 在 ES5 环境中可能需要调整为 `"es5"`

### 改进建议

1. **添加更多配置项**以提高一致性：
   ```json
   {
     "printWidth": 100,
     "singleQuote": false,
     "trailingComma": "all",
     "tabWidth": 2,
     "semi": true,
     "endOfLine": "lf"
   }
   ```

2. **考虑使用 `.prettierrc.js`**: 如果需要条件配置或注释，可以改用 JavaScript 格式

3. **与 EditorConfig 配合**: 建议添加 `.editorconfig` 文件，与 Prettier 配置保持一致：
   ```
   [*]
   indent_size = 2
   max_line_length = 100
   ```

4. **CI 集成**: 确保 `pnpm run format` 在 CI 中运行，防止未格式化的代码被合并
