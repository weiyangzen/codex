# codex-cli/.dockerignore 研究文档

## 场景与职责

`.dockerignore` 文件位于 `codex-cli/` 目录下，用于在构建 Docker 镜像时排除不需要的文件和目录。这是 Docker 构建上下文优化和安全加固的标准实践。

该文件服务于 `codex-cli/Dockerfile`，后者定义了 Codex CLI 的容器化运行环境。通过排除 `node_modules/`，可以避免将主机上的依赖目录复制到构建上下文中，从而减少构建时间、镜像大小和潜在的安全风险。

## 功能点目的

### 核心功能
- **排除 `node_modules/` 目录**：防止主机上的 Node.js 依赖被复制到 Docker 构建上下文

### 目的详解

1. **构建性能优化**
   - `node_modules/` 通常包含数千个文件，体积庞大
   - 排除后可显著减少 Docker 构建上下文大小，加快镜像构建速度

2. **依赖一致性保障**
   - Dockerfile 中通过 `npm install -g codex.tgz` 在容器内重新安装依赖
   - 避免主机与容器环境依赖版本不一致导致的"works on my machine"问题

3. **安全风险降低**
   - 防止主机上可能包含敏感信息（如 `.env` 文件、调试配置）的依赖目录被意外打包

## 具体技术实现

### 文件内容
```
node_modules/
```

### 技术细节
- **语法**：标准的 `.dockerignore` glob 模式
- **匹配规则**：排除所有名为 `node_modules` 的目录（无论在何层级）
- **作用域**：仅影响 Docker 构建上下文，不影响运行时文件系统

### 与 Dockerfile 的协作

在 `codex-cli/Dockerfile` 中：
```dockerfile
# Install codex
COPY dist/codex.tgz codex.tgz
RUN npm install -g codex.tgz
```

注意：
1. 构建时先执行 `pnpm pack` 生成 `codex.tgz`（在 `build_container.sh` 中完成）
2. 仅复制打包后的 `.tgz` 文件，而非整个源代码目录
3. 在容器内执行 `npm install` 安装依赖，确保环境纯净

## 关键代码路径与文件引用

### 直接关联文件
| 文件 | 关系 | 说明 |
|------|------|------|
| `codex-cli/Dockerfile` | 消费者 | 使用该 ignore 规则构建镜像 |
| `codex-cli/scripts/build_container.sh` | 构建脚本 |  orchestrates 镜像构建流程 |

### 构建流程
```
build_container.sh
  ├── pnpm install
  ├── pnpm run build
  ├── pnpm pack → dist/codex.tgz
  └── docker build -t codex -f Dockerfile .
      └── .dockerignore 生效，排除 node_modules/
```

## 依赖与外部交互

### 上游依赖
- Docker 引擎：解析 `.dockerignore` 规则
- `codex-cli/package.json`：定义了打包内容和入口点

### 下游影响
- Docker 构建上下文：排除后的文件不会被 `COPY` 指令访问
- 镜像层缓存：更小的上下文提高缓存命中率

## 风险、边界与改进建议

### 当前风险
1. **覆盖不足**：仅排除 `node_modules/`，未排除其他潜在敏感文件（如 `.git/`、`.env`、测试文件）
2. **隐式依赖**：如果 Dockerfile 未来改为直接复制源代码而非 `.tgz`，可能需要调整 ignore 规则

### 边界情况
- 该文件仅影响 `docker build` 命令，不影响 `docker run` 或容器运行时
- 如果通过 `-f` 指定其他路径的 Dockerfile，.dockerignore 仍从构建上下文根目录读取

### 改进建议

1. **扩展排除规则**（推荐）
   ```dockerignore
   node_modules/
   .git/
   .env*
   *.log
   .vscode/
   .idea/
   tests/
   docs/
   *.md
   ```

2. **添加注释说明**
   ```dockerignore
   # Dependencies are installed inside the container
   node_modules/
   ```

3. **与 .gitignore 对齐**
   - 当前 `.gitignore` 排除 `/vendor/`，但 `.dockerignore` 未排除
   - 如果 `vendor/` 包含构建产物，也应考虑排除

### 安全加固建议
- 考虑添加 `.git/` 排除，防止源代码历史泄露
- 考虑添加 `.env*` 排除，防止环境变量文件泄露
