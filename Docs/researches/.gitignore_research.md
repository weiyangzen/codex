# .gitignore 研究文档

## 场景与职责

`.gitignore` 是 Git 版本控制系统的核心配置文件，用于指定哪些文件和目录不应该被 Git 跟踪和提交到仓库。该文件位于项目根目录，定义了整个代码库的忽略规则，确保生成的文件、依赖目录、敏感信息等不会意外进入版本控制。

Codex 是一个多语言的 monorepo，包含 Rust、TypeScript/JavaScript、Nix 等组件，`.gitignore` 需要协调处理各种工具链和平台的生成文件。

## 功能点目的

### 1. 依赖管理

```gitignore
# Node.js dependencies
node_modules
.pnpm-store
.pnpm-debug.log
```

**目的**：排除 Node.js/npm/pnpm 的依赖目录和日志。

**技术细节**：
- `node_modules`：npm/yarn/pnpm 安装的依赖包目录
- `.pnpm-store`：pnpm 的本地存储目录（pnpm 使用内容寻址存储）
- `.pnpm-debug.log`：pnpm 调试日志

**例外规则**：
```gitignore
# Keep pnpm-lock.yaml
!pnpm-lock.yaml
```
- 锁定文件必须提交，确保依赖版本一致

### 2. 构建输出

```gitignore
# build
dist/
bazel-*
build/
out/
storybook-static/
```

**目的**：排除各种构建工具和框架的生成目录。

**详细说明**：

| 模式 | 来源 |
|------|------|
| `dist/` | TypeScript/JavaScript 项目默认输出 |
| `bazel-*` | Bazel 构建输出（bazel-bin, bazel-out, bazel-testlogs 等） |
| `build/` | 通用构建目录 |
| `out/` | VS Code 等工具默认输出 |
| `storybook-static/` | Storybook 静态构建输出 |

### 3. 发布相关

```gitignore
# ignore README for publishing
codex-cli/README.md

# ignore Nix derivation results
result
```

**目的**：
- `codex-cli/README.md`：CLI 包发布时可能有特殊的 README 处理
- `result`：Nix 构建结果（通常是到 `/nix/store` 的符号链接）

### 4. 编辑器配置

```gitignore
# editor
.vscode/
.idea/
.history/
.zed/
*.swp
*~
```

**目的**：排除编辑器/IDE 的配置和临时文件。

**编辑器覆盖**：
- `.vscode/`：VS Code 配置（但 `.vscode/` 目录本身可能有共享配置）
- `.idea/`：JetBrains IDE（WebStorm, IntelliJ, RustRover 等）
- `.history/`：某些编辑器的文件历史
- `.zed/`：Zed 编辑器配置
- `*.swp` / `*~`：Vim/Emacs 交换文件和备份

### 5. CLI 工具

```gitignore
# cli tools
CLAUDE.md
.claude/
AGENTS.override.md
```

**目的**：排除 AI 助手相关的本地文件。

**说明**：
- `CLAUDE.md` / `.claude/`：Claude AI 的本地上下文文件
- `AGENTS.override.md`：AGENTS.md 的本地覆盖文件

### 6. 缓存文件

```gitignore
# caches
.cache/
.turbo/
.parcel-cache/
.eslintcache
.nyc_output/
.jest/
*.tsbuildinfo
```

**目的**：排除各种工具生成的缓存。

**缓存类型**：
- `.cache/`：通用缓存目录
- `.turbo/`：Turborepo 缓存
- `.parcel-cache/`：Parcel 打包器缓存
- `.eslintcache`：ESLint 增量检查缓存
- `.nyc_output/`：NYC 代码覆盖率工具输出
- `.jest/`：Jest 缓存
- `*.tsbuildinfo`：TypeScript 增量编译信息

### 7. 日志文件

```gitignore
# logs
*.log
npm-debug.log*
yarn-debug.log*
yarn-error.log*
```

**目的**：排除运行时日志文件。

### 8. 环境变量

```gitignore
# env
.env*
!.env.example
```

**目的**：排除包含敏感信息的环境文件。

**安全考虑**：
- `.env` 通常包含 API 密钥、数据库密码等
- `.env.example` 提交作为模板（不含真实值）

### 9. 包管理器

```gitignore
# package
*.tgz

# Unwanted package managers
.yarn/
yarn.lock
```

**目的**：
- `*.tgz`：打包的 npm 包
- `.yarn/` / `yarn.lock`：项目使用 pnpm，排除 yarn 相关文件

### 10. CI/部署

```gitignore
# ci
.vercel/
.netlify/
```

**目的**：排除 Vercel 和 Netlify 的本地配置/缓存。

### 11. 补丁目录

```gitignore
# patches
apply_patch/
```

**目的**：排除临时补丁应用目录。

### 12. 代码覆盖率

```gitignore
# coverage
coverage/
```

**目的**：排除测试覆盖率报告输出。

### 13. 个人文件

```gitignore
# personal files
personal/
```

**目的**：排除个人工作目录。

### 14. 操作系统文件

```gitignore
# os
.DS_Store
Thumbs.db
Icon?
.Spotlight-V100/
```

**目的**：排除 macOS 和 Windows 的系统文件。

### 15. 发布产物

```gitignore
# release
package.json-e
session.ts-e
CHANGELOG.ignore.md
```

**目的**：排除发布过程中生成的临时文件。

**说明**：
- `package.json-e` / `session.ts-e`：sed 等工具生成的备份文件（`-e` 扩展名）
- `CHANGELOG.ignore.md`：可能用于过滤变更日志

### 16. Nix 相关

```gitignore
# nix related
.direnv
.envrc
```

**目的**：排除 Nix/direnv 的本地环境配置。

### 17. Python 字节码

```gitignore
# Python bytecode files
__pycache__/
*.pyc
```

**目的**：排除 Python 编译生成的字节码（项目可能包含 Python 脚本）。

## 具体技术实现

### Gitignore 语法

```gitignore
# 注释行
pattern/          # 匹配目录
pattern           # 匹配文件和目录
*.ext             # 通配符
!pattern          # 否定模式（不忽略）
**/pattern        # 任意层级的匹配
path/pattern      # 特定路径的匹配
```

### 模式优先级

```
1. 从工作区根目录开始读取 .gitignore
2. 子目录的 .gitignore 覆盖父目录规则
3. 后定义的规则覆盖先定义的规则
4. 否定模式（!）覆盖普通模式
```

### 与 .bazelignore 的关系

| 特性 | .gitignore | .bazelignore |
|------|-----------|--------------|
| 用途 | Git 版本控制 | Bazel 构建 |
| 语法 | 丰富（通配符、否定等） | 简单（仅路径列表） |
| 范围 | 递归生效 | 递归生效 |
| 目标 | 避免提交生成文件 | 避免扫描无关文件 |

**重叠内容**：
- 两者都排除 `node_modules`
- `.gitignore` 排除 `bazel-*`，而 `.bazelignore` 排除 `.git` 和 `codex-rs/target`

## 关键代码路径与文件引用

### 相关文件

1. **.bazelignore**
   - Bazel 的忽略配置
   - 与 `.gitignore` 互补

2. **package.json**
   - 定义 npm 脚本和依赖
   - `.gitignore` 排除 `node_modules` 但保留 `pnpm-lock.yaml`

3. **flake.nix** / **flake.lock**
   - Nix 配置
   - `.gitignore` 排除 `result`（构建输出）

4. **codex-rs/Cargo.toml**
   - Rust 配置
   - `.gitignore` 不直接排除 `target/`，因为 Bazel 构建可能使用不同目录

### 忽略规则生效流程

```
1. git status / git add 时读取 .gitignore
2. 对每个文件路径，从上到下匹配规则
3. 如果匹配普通模式 → 忽略
4. 如果匹配否定模式（!）→ 不忽略
5. 如果未匹配任何规则 → 跟踪
```

## 依赖与外部交互

### Git 工具

**核心命令**：
```bash
# 查看被忽略的文件
git status --ignored

# 检查某个文件是否被忽略
git check-ignore -v filename

# 强制添加被忽略的文件
git add -f filename

# 清理被忽略的文件（谨慎使用）
git clean -fdX
```

### 编辑器集成

大多数编辑器/IDE 会：
- 读取 `.gitignore` 决定哪些文件不显示在文件树中
- 使用灰色或隐藏图标标记被忽略的文件

### CI/CD 集成

```yaml
# 在 CI 中检查是否有应该被忽略的文件被提交
- run: |
    git check-ignore --no-index $(git ls-files) && exit 1 || true
```

## 风险、边界与改进建议

### 潜在风险

1. **过度忽略**
   - 某些配置可能过于宽泛
   - 例如：`.vscode/` 可能包含共享的调试配置

2. **忽略冲突**
   - `!.env.example` 确保模板被提交
   - 但如果有人创建 `.env.example.local`，也会被提交

3. **平台差异**
   - Windows 和 Unix 的路径分隔符差异
   - 某些模式可能在不同平台表现不同

4. **工具版本差异**
   - 新版本的工具可能使用不同的缓存目录
   - 需要定期更新 `.gitignore`

### 边界情况

1. **子模块**
   - 如果项目使用 Git 子模块，子模块有自己的 `.gitignore`
   - 父项目的 `.gitignore` 不影响子模块内部

2. **稀疏检出**
   - Git 稀疏检出模式下，`.gitignore` 行为可能不同

3. **符号链接**
   - 被忽略目录中的符号链接处理

### 改进建议

1. **分类和注释优化**
   ```gitignore
   # ============================================
   # Dependencies
   # ============================================
   node_modules/
   .pnpm-store/
   !pnpm-lock.yaml
   
   # ============================================
   # Build outputs
   # ============================================
   dist/
   build/
   out/
   ```

2. **添加更多工具支持**
   ```gitignore
   # Additional editors
   *.sublime-*
   .atom/
   
   # Additional tools
   .eslintcache*
   .prettiercache
   ```

3. **定期审查**
   ```bash
   # 查找被跟踪但可能被忽略的文件
   git ls-files | git check-ignore --stdin
   
   # 查找应该被忽略但未被忽略的文件
   find . -type f -name "*.log" | git check-ignore --stdin
   ```

4. **与 .bazelignore 同步**
   - 考虑将共同的忽略项文档化
   - 例如：两者都排除 `node_modules`

5. **敏感信息保护**
   ```gitignore
   # 添加更多安全相关模式
   *.pem
   *.key
   secrets/
   ```

### 使用示例

```bash
# 检查 .gitignore 是否有效
git check-ignore -v node_modules/package.json

# 查看所有被忽略的文件
git status --ignored

# 临时忽略 .gitignore（添加被忽略的文件）
git add -f codex-cli/README.md

# 从 Git 中移除但保留本地文件
git rm --cached codex-cli/README.md

# 清理所有被忽略的文件（⚠️ 危险）
git clean -fdX
```
