# .codespellrc 研究文档

## 场景与职责

`.codespellrc` 是 **codespell** 工具的主配置文件，用于定义拼写检查的行为、范围和例外规则。codespell 是一个专门用于检测源代码中常见拼写错误的工具，它通过匹配已知错误模式的数据库来工作，而不是使用传统的字典查找方法。

该文件位于项目根目录，与 `.codespellignore` 配合使用，共同确保代码库中的拼写质量，同时避免对专有名词、技术术语等产生误报。

## 功能点目的

### 1. 扫描范围配置

```ini
skip = .git*,vendor,*-lock.yaml,*.lock,.codespellrc,*test.ts,*.jsonl,frame*.txt,*.snap,*.snap.new,*meriyah.umd.min.js
```

**目的**：定义哪些文件或目录应该被跳过，不参与拼写检查。

**跳过模式详解**：

| 模式 | 说明 |
|------|------|
| `.git*` | Git 版本控制相关文件和目录 |
| `vendor` | 第三方依赖目录 |
| `*-lock.yaml` | 各种锁定文件（如 pnpm-lock.yaml） |
| `*.lock` | 锁文件 |
| `.codespellrc` | 配置文件本身 |
| `*test.ts` | TypeScript 测试文件 |
| `*.jsonl` | JSON Lines 格式文件（通常是数据文件） |
| `frame*.txt` | 帧数据文本文件 |
| `*.snap` | Jest/Insta 测试快照文件 |
| `*.snap.new` | 新生成的快照文件（待审查） |
| `*meriyah.umd.min.js` | 压缩后的 JavaScript 库文件 |

### 2. 隐藏文件检查

```ini
check-hidden = true
```

**目的**：启用对隐藏文件（以 `.` 开头的文件）的检查。

**技术背景**：
- 默认情况下，codespell 跳过隐藏文件
- 启用此选项确保 `.github/workflows/*.yml` 等隐藏目录中的文件也被检查
- 对于包含重要配置的项目很重要

### 3. 正则表达式忽略

```ini
ignore-regex = ^\s*"image/\S+": ".*|\b(afterAll)\b
```

**目的**：使用正则表达式定义应该忽略的模式。

**正则表达式解析**：

| 部分 | 含义 |
|------|------|
| `^\s*"image/\S+": ".*` | 匹配 JSON 中的 image 字段（如 `"image/png": "data:..."`） |
| `\|` | 或 |
| `\b(afterAll)\b` | 匹配单词 "afterAll"（Jest 测试钩子函数） |

**应用场景**：
- Base64 编码的图片数据可能包含看起来像拼写错误的字符序列
- `afterAll` 是 Jest 测试框架的标准函数名，不是拼写错误

### 4. 忽略词列表

```ini
ignore-words-list = ratatui,ser,iTerm,iterm2,iterm,te,TE
```

**目的**：定义应该被忽略的特定词汇列表。

**词汇详解**：

| 词汇 | 说明 |
|------|------|
| `ratatui` | Rust TUI 库名称（可能被误判为拼写错误） |
| `ser` | serde（序列化库）的缩写 |
| `iTerm` / `iterm2` / `iterm` | macOS 终端模拟器名称（各种大小写形式） |
| `te` / `TE` | 技术缩写（如 Terminal Emulator） |

**注意**：此列表与 `.codespellignore` 文件有重叠，两者共同作用。

## 具体技术实现

### 文件格式

`.codespellrc` 使用 INI 配置文件格式：

```ini
[codespell]
key = value
key2 = value2

[section2]
key = value
```

### 配置优先级

```
1. 命令行参数（最高优先级）
2. 当前目录的 .codespellrc
3. 上级目录的 .codespellrc
4. 用户主目录的 ~/.codespellrc
5. 默认配置（最低优先级）
```

### Codespell 工作流程

```
1. 读取配置文件
2. 解析 skip 模式，构建文件排除列表
3. 遍历文件系统，收集待检查文件
4. 对每个文件：
   a. 应用 ignore-regex 排除匹配行
   b. 检查剩余文本中的拼写错误
   c. 对照 ignore-words-list 过滤结果
5. 输出剩余的潜在拼写错误
```

## 关键代码路径与文件引用

### 相关文件

1. **.codespellignore**
   - 额外的忽略词列表
   - 与 `ignore-words-list` 功能相同，但维护方式不同
   - 包含：`iTerm`, `iTerm2`, `psuedo`, `te`, `TE`

2. **.github/workflows/codespell.yml**
   ```yaml
   - name: Codespell
     uses: codespell-project/actions-codespell@v2
     with:
       ignore_words_file: .codespellignore
   ```
   - CI 自动读取 `.codespellrc`
   - 额外指定 `.codespellignore` 文件

3. **AGENTS.md**
   - 项目文档，包含大量技术术语
   - 受益于拼写检查，确保专业文档质量

4. **docs/** 目录
   - 项目文档
   - 需要拼写检查保证专业性

### 扫描覆盖范围

根据配置，codespell 会检查：
- ✅ 所有源代码文件（`.rs`, `.ts`, `.js` 等）
- ✅ Markdown 文档（`.md`）
- ✅ 配置文件（`.yml`, `.yaml`, `.toml` 等）
- ✅ 隐藏目录中的文件（`.github/` 等）
- ❌ 生成/锁定文件（`*.lock`, `*-lock.yaml`）
- ❌ 测试快照（`*.snap`）
- ❌ 压缩的库文件（`*.min.js`）
- ❌ Git 目录

## 依赖与外部交互

### Codespell 工具

**安装方式**：
```bash
# pip
pip install codespell

# Homebrew
brew install codespell

# apt (Ubuntu)
sudo apt-get install codespell
```

**内置字典**：
- codespell 包含一个常见拼写错误的数据库
- 数据库格式：`错误拼写->正确拼写`
- 可以通过 `--dictionary` 添加自定义字典

### GitHub Actions 集成

```yaml
name: Codespell
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  codespell:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
      - uses: codespell-project/codespell-problem-matcher@v1
      - uses: codespell-project/actions-codespell@v2
        with:
          ignore_words_file: .codespellignore
```

**problem-matcher**：将 codespell 输出转换为 GitHub 注释，直接在 PR 中显示问题位置。

### 与 Pre-commit 集成

```yaml
# .pre-commit-config.yaml（如果项目使用）
repos:
  - repo: https://github.com/codespell-project/codespell
    rev: v2.2.6
    hooks:
      - id: codespell
        args: [--config, .codespellrc]
```

## 风险、边界与改进建议

### 潜在风险

1. **过度跳过**
   - `skip` 列表中的某些模式可能过于宽泛
   - 例如：`*test.ts` 跳过所有测试文件，可能遗漏真正的拼写错误

2. **正则表达式复杂性**
   - `ignore-regex` 复杂且难以维护
   - 建议添加注释解释每个部分的目的

3. **忽略词冲突**
   - `ser` 可能是 "serial" 的缩写，也可能是拼写错误
   - 需要平衡误报和漏报

4. **大小写敏感**
   - `iTerm` / `iterm` / `ITerm` 需要分别处理
   - 当前配置可能不完全覆盖所有变体

### 边界情况

1. **多语言代码库**
   - 项目包含 Rust、TypeScript、JavaScript 等
   - 每种语言可能有特定的术语需要处理

2. **生成的代码**
   - OpenAPI 生成的代码可能包含特定术语
   - 某些术语在生成代码中是合法的，但在手写代码中是错误

3. **第三方代码**
   - `vendor` 目录被跳过，但其他位置的第三方代码可能被检查

### 改进建议

1. **添加配置注释**
   ```ini
   [codespell]
   # Skip generated files, dependencies, and lock files
   skip = .git*,vendor,*-lock.yaml,*.lock,.codespellrc,*test.ts,*.jsonl,frame*.txt,*.snap,*.snap.new,*meriyah.umd.min.js
   
   # Check hidden files like .github/workflows
   check-hidden = true
   
   # Ignore: 1) JSON image data URLs, 2) Jest afterAll hook
   ignore-regex = ^\s*"image/\S+": ".*|\b(afterAll)\b
   
   # Technical terms and product names
   ignore-words-list = ratatui,ser,iTerm,iterm2,iterm,te,TE
   ```

2. **优化 skip 列表**
   ```ini
   # 考虑添加：
   skip = ...,*.gen.ts,*.generated.rs,...
   # 生成的 TypeScript/Rust 文件
   ```

3. **统一忽略词管理**
   - 当前同时使用 `ignore-words-list` 和 `.codespellignore`
   - 建议：将所有忽略词移到 `.codespellignore`，每行一个
   - 或：完全使用 `ignore-words-list`，删除 `.codespellignore`

4. **定期审查 ignore-regex**
   ```bash
   # 检查当前正则是否仍然需要
   grep -r "afterAll" --include="*.ts" --include="*.js" .
   # 如果没有使用 Jest，可以移除 afterAll 的忽略
   ```

5. **添加更多技术术语**
   ```ini
   # 根据项目实际使用的技术栈
   ignore-words-list = ...,tokio,serde,clippy,rustfmt,...
   ```

6. **考虑大小写不敏感**
   ```ini
   # 如果 codespell 支持，启用大小写不敏感匹配
   # 减少重复条目如 te/TE, iterm/iTerm
   ```

### 使用示例

```bash
# 基本使用（自动读取 .codespellrc）
codespell

# 检查特定目录
codespell codex-rs/src

# 交互式修复
codespell --interactive=3

# 显示将被忽略的内容（调试用）
codespell --debug

# 写入修复（谨慎使用）
codespell --write-changes

# 忽略配置，强制检查所有
codespell --skip="" --ignore-words-list=""
```

### 相关配置对比

| 配置项 | 位置 | 用途 |
|--------|------|------|
| `skip` | `.codespellrc` | 跳过整个文件/目录 |
| `ignore-regex` | `.codespellrc` | 跳过匹配的行/模式 |
| `ignore-words-list` | `.codespellrc` | 忽略特定词汇 |
| `.codespellignore` | 单独文件 | 额外的忽略词列表 |

**最佳实践建议**：
- 使用 `skip` 排除生成文件和依赖
- 使用 `ignore-regex` 排除特定格式（如 Base64 数据）
- 使用 `.codespellignore` 维护技术术语列表（便于版本控制审查）
- 避免在 `ignore-words-list` 和 `.codespellignore` 中重复
