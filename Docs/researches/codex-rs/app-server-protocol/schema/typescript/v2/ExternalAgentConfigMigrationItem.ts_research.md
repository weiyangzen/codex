# ExternalAgentConfigMigrationItem.ts Research Document

## 场景与职责

`ExternalAgentConfigMigrationItem` 是配置迁移功能的核心数据类型，代表一个可迁移的外部 Agent 配置项。它封装了从其他 AI 编码工具（如 Claude、Cursor 等）迁移配置所需的所有元数据，包括配置类型、描述信息和作用范围。

该类型在配置迁移流程中被广泛使用：
- 作为 `ExternalAgentConfigDetectResponse` 的数组元素，表示检测到的可迁移配置
- 作为 `ExternalAgentConfigImportParams` 的数组元素，表示用户选择要导入的配置

## 功能点目的

1. **配置标识**: 唯一标识一个可迁移的配置资源
2. **类型分类**: 区分不同类型的配置（AGENTS_MD、CONFIG、SKILLS、MCP_SERVER_CONFIG）
3. **范围界定**: 明确配置是全局（home-scoped）还是项目级（repo-scoped）
4. **用户展示**: 提供人类可读的描述信息，帮助用户理解配置内容

## 风险、边界与改进建议

### 潜在风险

1. **路径安全问题**: `cwd` 字段包含文件系统路径，可能涉及敏感信息泄露
2. **描述注入**: `description` 字段如果包含未过滤的用户输入，可能在 UI 中造成 XSS 风险
3. **类型误匹配**: `itemType` 与实际配置内容不匹配可能导致导入失败或配置损坏
4. **路径遍历**: 如果 `cwd` 包含恶意构造的路径（如 `../../../etc`），可能导致安全风险

### 边界情况

1. **空描述**: `description` 为空字符串时，客户端需要准备默认展示文本
2. **无效路径**: `cwd` 指向的目录在导入时可能已被删除或移动
3. **重复项**: 同一个物理配置可能因符号链接等原因被多次检测
4. **跨平台路径**: Windows 和 Unix 风格的路径格式差异需要处理
