# FsReadDirectoryResponse.ts Research Document

## 场景与职责

`FsReadDirectoryResponse` 是 Codex App-Server Protocol v2 中文件系统操作 API 的一部分，专门用于响应 `fs/readDirectory` RPC 方法调用。该类型在客户端需要浏览主机文件系统目录结构时发挥关键作用，例如：

- IDE 或编辑器插件需要展示项目文件树
- 文件浏览器组件需要列出目录内容
- 自动化工具需要遍历目录以发现特定文件
- 智能体(agent)需要探索工作空间以理解项目结构

作为 App-Server 架构中的响应类型，它封装了目录读取操作的结果，为客户端提供标准化的目录条目信息。

## 功能点目的

该类型的核心目的是：

1. **标准化目录列表响应格式**：为 `fs/readDirectory` 操作提供统一的响应结构
2. **提供目录条目元数据**：返回目录中每个子项的基本信息（名称、类型）
3. **支持文件系统导航**：使客户端能够递归遍历文件系统或构建文件树视图
4. **类型安全保障**：通过 TypeScript 类型系统确保响应数据的结构一致性

与直接使用底层文件系统 API 不同，该类型通过 App-Server 协议层提供跨平台、语言无关的目录访问抽象。

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `entries` | `Array<FsReadDirectoryEntry>` | 是 | 请求目录的直接子条目数组。每个条目包含文件名、是否为目录、是否为文件等元数据 |

**FsReadDirectoryEntry 结构详情：**

```typescript
export type FsReadDirectoryEntry = { 
  /**
   * Direct child entry name only, not an absolute or relative path.
   */
  fileName: string, 
  /**
   * Whether this entry resolves to a directory.
   */
  isDirectory: boolean, 
  /**
   * Whether this entry resolves to a regular file.
   */
  isFile: boolean, 
};
```

- `fileName`: 仅包含条目名称（如 `"src"`、`"package.json"`），不含路径信息
- `isDirectory`: 布尔值，标识该条目是否为目录
- `isFile`: 布尔值，标识该条目是否为常规文件

**注意事项：**
- 返回的条目仅包含直接子项，不包含嵌套子目录的内容
- 条目不包含隐藏文件过滤逻辑，由客户端决定如何处理
- 不包含文件大小、修改时间等详细元数据，如需更多信息需使用 `fs/readFile` 或其他 API

## 风险、边界与改进建议

### 潜在风险

1. **大目录性能问题**：如果目录包含大量条目（如 `node_modules`），返回的数组可能很大，影响序列化和传输性能
2. **符号链接处理**：当前类型定义不明确说明符号链接的处理方式（跟随链接还是返回链接本身）
3. **权限错误**：当客户端无权限读取目录时，错误处理依赖于底层实现，可能需要明确的错误类型

### 边界情况

| 场景 | 预期行为 |
|------|----------|
| 空目录 | 返回 `{ entries: [] }` |
| 目录不存在 | 应返回错误而非空数组 |
| 路径指向文件而非目录 | 应返回错误 |
| 符号链接 | 行为未明确，可能跟随或返回链接元数据 |

### 改进建议

1. **分页支持**：对于可能包含大量条目的目录，考虑添加分页参数（cursor/limit）
   ```typescript
   export type FsReadDirectoryResponse = { 
     entries: Array<FsReadDirectoryEntry>,
     nextCursor?: string | null,  // 建议添加
   };
   ```

2. **增强元数据**：考虑添加更多文件元数据（大小、修改时间、权限）以减少后续 API 调用

3. **过滤选项**：在请求参数中添加通配符或正则过滤，减少不必要的数据传输

4. **错误类型细化**：定义明确的错误类型（如 `DirectoryNotFound`、`PermissionDenied`）

5. **符号链接明确化**：添加 `isSymlink` 字段以明确标识符号链接

### 版本兼容性

- 该类型属于 App-Server Protocol v2，与 v1 不兼容
- 代码由 `ts-rs` 从 Rust 源自动生成，手动修改会被覆盖
- 如需修改，应更新 `codex-rs/app-server-protocol/src/protocol/v2.rs` 中的 Rust 定义
