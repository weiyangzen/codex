# FsRemoveParams.ts Research Document

## 场景与职责

`FsRemoveParams` 是 Codex App-Server Protocol v2 中文件系统删除操作的请求参数类型，用于 `fs/remove` RPC 方法。该类型在以下场景中发挥关键作用：

- **文件清理**：删除临时文件、缓存文件或不再需要的资源
- **目录清理**：删除整个目录树（如清理构建输出 `node_modules`、`target` 等）
- **项目重置**：删除并重置项目状态
- **自动化清理**：CI/CD 流程中的清理步骤
- **智能体操作**：AI 智能体执行代码重构时删除旧文件

作为 App-Server 架构的请求参数类型，它提供了灵活且安全的文件/目录删除控制机制。

## 功能点目的

该类型的核心设计目标：

1. **统一删除接口**：单一接口处理文件和目录删除
2. **递归控制**：通过 `recursive` 参数控制是否删除目录内容
3. **容错能力**：通过 `force` 参数控制是否忽略不存在的路径
4. **安全默认**：合理的默认值减少意外数据丢失

设计哲学：
- **幂等性**：`force: true` 时多次调用结果一致（不报错）
- **显式控制**：递归删除需要显式启用（防止意外）
- **路径安全**：使用 `AbsolutePathBuf` 确保路径明确

  /**
   * Whether directory removal should recurse. Defaults to `true`.
   */

   */
  force?: boolean | null,

### 关键字段说明

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| `path` | `AbsolutePathBuf` | 是 | - | 要删除的文件或目录的绝对路径 |
| `recursive` | `boolean \| null` | 否 | `true` | 是否递归删除目录内容 |
| `force` | `boolean \| null` | 否 | `true` | 是否忽略不存在的路径 |

**字段详解：**

#### `path`
- 必须是绝对路径
- 可以是文件或目录
- 受沙箱权限控制

#### `recursive`
- `true`（默认）：递归删除目录及其所有内容
- `false`：仅删除空目录，非空目录会报错
- `null`：使用默认值 `true`

#### `force`
- `true`（默认）：如果路径不存在，静默成功
- `false`：如果路径不存在，返回错误
- `null`：使用默认值 `true`

**使用模式：**

```typescript
// 安全删除文件（force: true 确保幂等）
{ path: "/workspace/temp.txt" }

// 删除目录及其所有内容
{ path: "/workspace/build", recursive: true }

// 仅删除空目录
{ path: "/workspace/empty-dir", recursive: false }

// 严格模式：路径必须存在
{ path: "/workspace/important.txt", force: false }
```

## 风险、边界与改进建议

### 潜在风险

1. **数据丢失风险**：删除操作不可逆，误删可能导致数据永久丢失
2. **递归删除危险**：`recursive: true` 可能意外删除大量数据
3. **符号链接处理**：删除符号链接时的行为需要明确（删除链接本身还是目标）
4. **并发问题**：删除过程中文件被其他进程修改或锁定

### 安全建议

1. **沙箱限制**：确保只能在允许的工作空间内删除
2. **重要文件保护**：对关键配置文件添加保护机制
3. **操作确认**：对于大范围删除，建议客户端要求用户确认
4. **回收站机制**：考虑实现软删除/回收站功能

### 边界情况

| 场景 | 预期行为 |
|------|----------|
| 删除根目录 `/` | 应被拒绝（安全限制） |
| 删除当前工作目录 | 可能成功，但后续操作可能失败 |
| 删除只读文件 | 取决于文件系统权限 |
| 删除被占用的文件 | 取决于操作系统（Windows 通常失败） |
| 符号链接 | 删除链接本身，不删除目标 |
| 硬链接 | 减少链接计数，文件可能仍然存在 |

### 改进建议

1. **添加 dryRun 选项**：预览将要删除的内容
   ```typescript
   export type FsRemoveParams = { 
     path: AbsolutePathBuf,
     recursive?: boolean | null,
     force?: boolean | null,
     dryRun?: boolean | null,  // 建议添加
   };
   ```

2. **添加 include/exclude 模式**：细粒度控制删除范围
   ```typescript
   export type FsRemoveParams = { 
     path: AbsolutePathBuf,
     recursive?: boolean | null,
     include?: string[],  // glob 模式
     exclude?: string[],  // glob 模式
   };
   ```

3. **返回删除统计**：告知客户端实际删除了什么
   ```typescript
   export type FsRemoveResponse = {
     deletedFiles: number,
     deletedDirectories: number,
     bytesFreed: number,
   };
   ```

4. **软删除支持**：移动到回收站而非永久删除
   ```typescript
   export type FsRemoveParams = { 
     path: AbsolutePathBuf,
     permanent?: boolean | null,  // false = 移动到回收站
   };
   ```

5. **批量删除**：支持一次删除多个路径
   ```typescript
   export type FsRemoveBatchParams = {
     items: Array<{ path: AbsolutePathBuf, recursive?: boolean }>,
   };
   ```
