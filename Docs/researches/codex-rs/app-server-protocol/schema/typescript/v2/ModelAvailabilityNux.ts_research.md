# ModelAvailabilityNux.ts 调研文档

## 场景与职责

`ModelAvailabilityNux` 是 Codex App Server Protocol v2 API 中用于表示模型可用性新用户体验（NUX - New User Experience）消息的数据结构。当某个模型对用户变得可用时，系统可以通过此结构向客户端展示特定的引导消息或通知。

主要使用场景包括：
- 当用户获得访问某个新模型的权限时，展示引导性消息
- 在模型列表中标记需要特殊说明的模型
- 向用户传达关于模型可用性的重要信息

## 功能点目的

该类型的核心目的是提供一个标准化的方式来传递模型可用性相关的用户消息：

1. **消息传递**：通过 `message` 字段向用户展示关于模型可用性的文本信息
2. **用户体验增强**：在用户首次获得模型访问权限时提供上下文说明
3. **模型发现**：帮助用户了解新可用模型的特性和用途

TypeScript 定义：
```typescript
export type ModelAvailabilityNux = { message: string }
```

## 风险、边界与改进建议

### 潜在风险

1. **消息长度无限制**：`message` 字段没有长度限制，过长的消息可能影响 UI 展示
   - 建议：在客户端实现消息截断或折叠机制

2. **国际化支持**：当前 `message` 是单一字符串，缺乏多语言支持
   - 风险：非英语用户可能无法理解消息内容
   - 建议：考虑添加 i18n 键值或本地化标识

3. **XSS 风险**：如果 `message` 包含 HTML/JS 内容且客户端直接渲染，可能存在安全风险
   - 建议：客户端应始终对消息内容进行转义处理

### 边界情况

1. **空消息**：`message` 为空字符串时的展示处理
2. **特殊字符**：消息中包含换行符、Unicode 字符等的渲染
3. **并发更新**：模型可用性状态变化时的消息同步

### 改进建议

1. **添加元数据字段**：
   ```rust
   pub struct ModelAvailabilityNux {
       pub message: String,
       pub severity: Option<NuxSeverity>,  // info, warning, etc.
       pub action_url: Option<String>,     // 可点击的链接
       pub expires_at: Option<i64>,        // 消息过期时间
   }
   ```

2. **支持 Markdown**：允许 `message` 使用 Markdown 格式，增强表达能力

3. **添加消息 ID**：便于客户端追踪已展示的消息，避免重复显示

4. **条件展示**：添加条件字段控制消息的展示时机（如首次使用、每次访问等）

5. **测试覆盖**：
   - 添加序列化/反序列化测试
   - 验证与核心协议层的转换逻辑
   - 测试特殊字符和边界长度
