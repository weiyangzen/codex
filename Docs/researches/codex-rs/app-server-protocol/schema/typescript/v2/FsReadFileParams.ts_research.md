# FsReadFileParams.ts Research Document

## 场景与职责

`FsReadFileParams` 是 Codex App-Server Protocol v2 中文件系统读取操作的请求参数类型，用于 `fs/readFile` RPC 方法。该类型在以下场景中发挥作用：

- **文件内容获取**：客户端需要读取主机上的文件内容以进行显示或处理
- **代码分析工具**：智能体需要读取源代码文件进行分析和理解
- **配置读取**：应用需要读取配置文件（如 `config.toml`、`.env` 等）
- **资源加载**：加载项目资源文件（图片、数据文件等）
- **日志查看**：读取日志文件进行调试和监控

作为 App-Server 架构的请求参数类型，它定义了读取文件操作所需的最小信息集，确保调用方提供明确的文件路径标识。

## 功能点目的

该类型的核心设计目标：

1. **路径明确性**：要求使用绝对路径，消除相对路径解析的歧义性
2. **安全控制**：通过 `AbsolutePathBuf` 类型确保路径经过验证和规范化
3. **最小化参数**：仅包含必要信息（路径），保持接口简洁
4. **跨平台兼容**：抽象不同操作系统的路径差异

与直接使用文件系统 API 相比，该类型通过协议层提供：
- 统一的路径验证（防止路径遍历攻击）
- 沙箱边界控制（限制可访问的文件范围）
- 序列化/反序列化的类型安全

**AbsolutePathBuf 类型详情：**

```typescript
/**
 * A path that is guaranteed to be absolute and normalized (though it is not
 * guaranteed to be canonicalized or exist on the filesystem).
 *
 * IMPORTANT: When deserializing an `AbsolutePathBuf`, a base path must be set
 * using [AbsolutePathBufGuard::new]. If no base path is set, the
 * deserialization will fail unless the path being deserialized is already
 * absolute.
 */
export type AbsolutePathBuf = string;
```

**关键特性：**
- 必须是绝对路径（以 `/` 开头在 Unix，或以盘符开头在 Windows）
- 路径会被规范化（消除 `.` 和 `..` 组件）
- 不保证路径存在或可被访问（需在运行时验证）
- 不保证是规范路径（canonical path，即不包含符号链接）

**路径示例：**
- ✅ 有效: `/home/user/project/src/main.rs`
- ✅ 有效: `/workspace/config.toml`
- ❌ 无效: `./src/main.rs`（相对路径）
- ❌ 无效: `../config.toml`（相对路径）

### 错误处理

可能的错误场景（由服务器返回）：
- 文件不存在
- 权限不足
- 路径不是文件（是目录）
- 路径超出沙箱允许范围
- 文件过大导致内存问题

## 风险、边界与改进建议

### 潜在风险

1. **路径遍历攻击**：尽管使用 `AbsolutePathBuf`，仍需确保服务器端验证路径在允许范围内
2. **大文件读取**：没有文件大小限制参数，可能导致内存问题
3. **二进制文件处理**：所有内容以 Base64 返回，对于大文件会增加约 33% 的传输开销
4. **并发读取**：没有并发控制机制，大量并发读取可能影响服务器性能

### 边界情况

| 场景 | 预期行为 |
|------|----------|
| 文件不存在 | 返回明确的错误 |
| 路径是目录 | 返回错误（应使用 `fs/readDirectory`） |
| 空文件 | 返回 `{ dataBase64: "" }` |
| 符号链接 | 跟随链接读取目标文件内容 |
| 权限不足 | 返回权限错误 |

### 改进建议

1. **添加范围读取支持**：支持读取文件的特定字节范围
   ```typescript
   export type FsReadFileParams = { 
     path: AbsolutePathBuf,
     offset?: number | null,  // 建议添加
     length?: number | null,  // 建议添加
   };
   ```

2. **添加编码选项**：允许指定返回编码（Base64、UTF-8、原始二进制）
   ```typescript
   export type FsReadFileParams = { 
     path: AbsolutePathBuf,
     encoding?: 'base64' | 'utf8' | 'binary' | null,
   };
   ```

3. **添加大小限制**：防止意外读取超大文件
   ```typescript
   export type FsReadFileParams = { 
     path: AbsolutePathBuf,
     maxSize?: number | null,
   };
   ```

4. **流式读取支持**：对于大文件，支持分块流式返回

5. **缓存控制**：添加缓存相关头部，支持条件读取（If-Modified-Since 等）

### 版本兼容性

- 该类型属于 App-Server Protocol v2
- 与 v1 的类似功能可能不兼容
- 代码由 `ts-rs` 自动生成，修改 Rust 源文件后需重新生成

### 安全考虑

1. **路径验证**：服务器必须验证路径在允许的工作空间范围内
2. **资源限制**：应设置文件大小上限和读取频率限制
3. **审计日志**：敏感文件读取应记录审计日志
4. **沙箱隔离**：确保不同会话间的文件访问隔离
