# prompt_for_init_command.md 研究文档

## 场景与职责

`prompt_for_init_command.md` 是 Codex TUI 中 `/init` 斜杠命令的默认提示词模板。当用户在 Codex CLI 中执行 `/init` 命令时，该文件内容被读取并作为系统提示词发送给 AI 模型，指导其生成项目的 `AGENTS.md` 文件。

## 功能点目的

1. **标准化 AGENTS.md 生成**：确保生成的贡献者指南具有一致的结构和风格
2. **提供结构化指导**：通过明确的章节大纲引导 AI 生成高质量文档
3. **定义内容约束**：规定字数限制（200-400 词）、格式要求（Markdown 标题）
4. **确保实用性**：要求包含具体的命令示例、路径和命名模式

## 具体技术实现

### 文件加载方式

在 `src/chatwidget.rs` 中通过 `include_str!` 宏在编译时嵌入：

```rust
// src/chatwidget.rs
const INIT_PROMPT: &str = include_str!("../prompt_for_init_command.md");

// 在 /init 命令处理中
SlashCommand::Init => {
    if self.project_doc_exists() {
        self.add_info_message(
            "{DEFAULT_PROJECT_DOC_FILENAME} already exists here. Skipping /init to avoid overwriting it."
        );
        return;
    }
    self.submit_user_message(INIT_PROMPT.to_string().into());
}
```

## 风险、边界与改进建议

### 潜在风险

1. **提示词注入风险**：用户可能通过项目文件影响 AI 生成的 AGENTS.md 内容
2. **版本漂移**：提示词模板更新后，旧版本 Codex 生成的 AGENTS.md 格式不一致
3. **语言限制**：当前模板为英文，非英语项目可能需要本地化

### 边界条件

1. **文件大小限制**：`include_str!` 有编译时大小限制（通常数 MB）
2. **编码要求**：必须是有效 UTF-8，否则编译失败
3. **路径依赖**：编译时相对路径 `../prompt_for_init_command.md` 必须存在
