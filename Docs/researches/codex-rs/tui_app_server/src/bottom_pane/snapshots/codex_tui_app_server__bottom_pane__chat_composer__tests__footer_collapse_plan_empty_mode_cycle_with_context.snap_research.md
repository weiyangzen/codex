# Footer Collapse Test: Plan Mode Empty Cycle With Context

## 1. 场景与职责 (Scenario and Responsibility)

This snapshot test validates the footer rendering behavior in the `tui_app_server` crate's ChatComposer component when:
- The composer textarea is **empty** (idle state)
- The terminal width is **60 columns** (medium width)
- **Plan mode** collaboration mode is active
- Context indicator (`"100% context left"`) is **visible**
- Mode cycle hint (`"(shift+tab to cycle)"`) is shown

This test verifies the footer maintains full functionality at medium widths with collaboration mode active.

## 2. 功能点目的 (Feature Purpose)

The test verifies the **medium-width layout** with Plan mode active. At 60 columns:
- Shows `"Plan mode (shift+tab to cycle)"` in magenta
- Shows `"100% context left"` context indicator
- The shortcuts hint is **dropped** to make room for the mode indicator
- Mode indicator takes priority over generic shortcuts hint

This demonstrates the priority system when space is constrained: mode indicator > shortcuts hint > context.

## 3. 具体技术实现 (Technical Implementation)

### Test Setup
```rust

    "footer_collapse_plan_empty_mode_cycle_with_context",
    60,  // medium width
    true, // enhanced_keys_supported

        composer.set_collaboration_modes_enabled(true);
        composer.set_collaboration_mode_indicator(Some(CollaborationModeIndicator::Plan));
        composer.set_context_window(Some(100), None);
    },

```

### Footer Rendering Flow
1. `single_line_footer_layout()` evaluates layout options:
   - Default with shortcuts hint: would fit but mode indicator is priority
   - With `collaboration_mode_indicator.is_some()` and `show_cycle_hint = true`:
     - First tries mode with cycle hint + context
     - At 60 columns, `"Plan mode (shift+tab to cycle)"` (~32 chars) + context (~17) + gap fits
2. Returns mode-only line with context

## 6. 风险、边界与改进建议 (Risks, Edge Cases, and Improvement Suggestions)

### Risks and Edge Cases
1. **Mode Label Truncation**: Long mode names could cause issues at this width
2. **Cycle Hint Length**: "(shift+tab to cycle)" is 20 characters - quite long
3. **Width Precision**: 60 columns is carefully chosen; 59 might drop context

### Boundary Conditions
- At 120 columns: Full display with shortcuts hint (`footer_collapse_plan_empty_full`)
- At 60 columns: This test - mode with cycle hint + context
- At 44 columns: Mode with cycle hint, no context (`footer_collapse_plan_empty_mode_cycle_without_context`)
- At 26 columns: Mode only (`footer_collapse_plan_empty_mode_only`)

### Improvement Suggestions
1. **Abbreviated Cycle Hint**: Consider "(Shift+Tab)" or "↹" symbol to save space
2. **Smart Truncation**: Truncate cycle hint before hiding context
3. **Configurable Priority**: Allow users to prioritize context over mode indicator
4. **Dynamic Sizing**: Measure actual rendered width instead of estimating

### Related Tests
- `footer_collapse_plan_empty_full` - 120 columns, all elements
- `footer_collapse_plan_empty_mode_cycle_without_context` - 44 columns, no context
- `footer_collapse_plan_empty_mode_only` - 26 columns, mode only

### Comparison with Non-Plan Mode
- Without mode indicator at 60 cols: Shows shortcuts hint + context
- With Plan mode at 60 cols: Shows mode + context (this test)
- Mode indicator replaces shortcuts hint, not added to it
