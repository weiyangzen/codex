# 快照分析：image_placeholder_multiple

## 场景与职责

该快照展示了 ChatComposer 处理多个图片附件时的 UI 渲染效果。当用户粘贴或选择多张图片时，界面需要清晰地展示所有已附加的图片，同时保持输入区域的可用性。

## 功能点目的

- **多图片占位符显示**：同时显示 `[Image #1]` 和 `[Image #2]`，表明有两张图片已附加
- **上下文余量提示**：显示 "100% context left"，告知用户当前输入占用的上下文比例
- **视觉层级**：通过编号区分不同的图片附件，便于用户识别和管理

## 具体技术实现

- 使用 `[Image #{index}]` 格式为每张图片生成占位符文本
- 占位符按顺序排列，索引从 1 开始递增
- 上下文余量计算基于当前输入内容（包括占位符）与最大上下文限制的比例

## 关键代码路径与文件引用

- `codex-rs/tui_app_server/src/bottom_pane/chat_composer.rs` - ChatComposer 组件实现
- 图片占位符渲染逻辑
- 上下文使用量计算与显示

## 依赖与外部交互

- 依赖图片附件列表状态管理
- 与上下文长度计算模块交互

## 风险、边界与改进建议

- **边界**：大量图片时占位符可能占用过多显示空间
- **改进建议**：考虑在图片数量较多时折叠显示或提供缩略图预览
