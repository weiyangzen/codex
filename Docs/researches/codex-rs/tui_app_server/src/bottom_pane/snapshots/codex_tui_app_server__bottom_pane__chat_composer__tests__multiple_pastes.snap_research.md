# 快照分析：multiple_pastes

## 场景与职责

该快照展示了 ChatComposer 处理多次粘贴操作时的混合显示效果。当用户连续粘贴多段内容时，系统需要正确区分和管理每段粘贴内容。

## 功能点目的

- **多段粘贴管理**：显示 `[Pasted Content 1003 chars]` 和 `[Pasted Content 1007 chars]` 两个大段粘贴占位符
- **混合内容显示**：在占位符之间或之后可以输入普通文本 "another short paste"
- **顺序保持**：保持粘贴操作的先后顺序

## 具体技术实现

- 每次粘贴操作独立评估是否超过 `LARGE_PASTE_CHAR_THRESHOLD`
- 超过阈值的内容转换为占位符，否则保留原文
- 占位符与普通文本在编辑器中按插入顺序排列
- 每段粘贴内容的长度独立计算并显示在占位符中

## 关键代码路径与文件引用

- `codex-rs/tui_app_server/src/bottom_pane/chat_composer.rs` - ChatComposer 组件实现
- 多次粘贴事件处理逻辑
- 内容分段管理

## 依赖与外部交互

- 依赖剪贴板访问
- 与内容分段存储系统交互

## 风险、边界与改进建议

- **边界**：
  - 多次粘贴后界面可能变得复杂
  - 用户可能难以区分不同粘贴段
- **改进建议**：
  - 为每个粘贴段添加时间戳或序号标识
  - 支持单独删除某段粘贴内容
  - 提供粘贴历史管理功能
