# Footer Collapse Test: Empty Mode Cycle With Context

## 1. 场景与职责 (Scenario and Responsibility)

This snapshot test validates the footer rendering behavior in the `tui_app_server` crate's ChatComposer component when:
- The composer textarea is **empty** (idle state)
- The terminal width is **60 columns** (medium width)
- Context window percentage is displayed (**100% context left**)
- Collaboration modes are enabled but **no specific mode indicator** is active

This test ensures the footer properly displays the shortcuts hint with context information at medium terminal widths.

## 2. 功能点目的 (Feature Purpose)

The test verifies the **footer collapse behavior** - specifically how the footer adapts its content when space is constrained. At 60 columns width:
- The footer should show `"? for shortcuts"` hint
- The context indicator (`"100% context left"`) should be visible on the right side
- Both elements should fit comfortably without truncation

This is part of the responsive footer design that provides users with contextual help while displaying system status.

## 3. 具体技术实现 (Technical Implementation)

### Test Setup
```rust
snapshot_composer_state_with_width(
    "footer_collapse_empty_mode_cycle_with_context",
    60,  // width
    true, // enhanced_keys_supported
    |composer| {
        composer.set_collaboration_modes_enabled(true);
        composer.set_collaboration_mode_indicator(None); // No mode indicator
        composer.set_context_window(Some(100), None); // 100% context
    },
);
```

### Footer Rendering Flow
1. `ChatComposer::render()` calls `footer_props()` to gather footer state
2. `footer_height()` calculates required footer lines based on `FooterProps`
3. `single_line_footer_layout()` determines the best layout for the available width:
   - Evaluates if full hint + context can fit
   - At 60 columns: `"? for shortcuts"` (~18 chars) + `"100% context left"` (~17 chars) + gap fits
4. `render_footer_from_props()` renders the final footer line

### Key Logic in `single_line_footer_layout()`
- `can_show_left_with_context()` checks if both left hint and right context fit
- Default state uses `SummaryHintKind::Shortcuts` for empty composer
- Context line is generated by `context_window_line(100, None)`

## 6. 风险、边界与改进建议 (Risks, Edge Cases, and Improvement Suggestions)

### Risks and Edge Cases
1. **Width Sensitivity**: The test uses exactly 60 columns; small changes to hint text could break layout
2. **Context Percentage Formatting**: If context formatting changes (e.g., "100%" → "Full"), snapshot will mismatch
3. **Translation Impact**: If hints become translatable, text length variations could affect collapse behavior

### Boundary Conditions
- **Minimum width**: At 26 columns, the footer drops to mode-only display (see `footer_collapse_empty_mode_only`)
- **Maximum width**: At 120 columns, full display with all elements (see `footer_collapse_empty_full`)
- **Without context**: At 44 columns, context is hidden (see `footer_collapse_empty_mode_cycle_without_context`)

### Improvement Suggestions
1. **Parameterized Testing**: Consider using test-case parameterized tests for width boundaries
2. **Visual Regression**: Add ASCII-art diagrams in comments showing expected layout
3. **Accessibility**: Ensure color-only indicators (like mode colors) have text alternatives
4. **Dynamic Width Testing**: Test behavior at width transitions (e.g., exactly where context gets hidden)

### Related Tests
- `footer_collapse_empty_full` - Wide terminal (120 cols)
- `footer_collapse_empty_mode_cycle_without_context` - Narrower terminal (44 cols)
- `footer_collapse_empty_mode_only` - Very narrow terminal (26 cols)
