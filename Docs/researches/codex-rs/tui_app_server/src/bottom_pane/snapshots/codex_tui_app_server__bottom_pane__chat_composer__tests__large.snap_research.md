# 快照分析：large

## 场景与职责

该快照展示了 ChatComposer 处理大段粘贴内容时的优化显示策略。当用户粘贴超过阈值（1000 字符）的内容时，系统使用占位符替代完整文本显示，以节省界面空间。

## 功能点目的

- **大内容占位符**：显示 `[Pasted Content 1005 chars]` 代替实际 1005 字符的内容
- **上下文余量提示**：显示 "100% context left"，尽管内容较长但仍占用较少上下文比例
- **界面优化**：避免大段文本直接渲染导致的界面拥挤

## 具体技术实现

- 定义常量 `LARGE_PASTE_CHAR_THRESHOLD = 1000` 作为阈值
- 超过阈值时生成 `[Pasted Content {n} chars]` 格式的占位符
- 实际内容仍被保留并发送至后端，仅 UI 显示被优化

## 关键代码路径与文件引用

- `codex-rs/tui_app_server/src/bottom_pane/chat_composer.rs` - ChatComposer 组件实现
- 粘贴内容处理与占位符生成逻辑

## 依赖与外部交互

- 依赖粘贴事件处理
- 与内容长度计算模块交互

## 风险、边界与改进建议

- **边界**：阈值 1000 字符是经验值，可能需要根据屏幕尺寸动态调整
- **改进建议**：
  - 提供展开/折叠功能，允许用户查看完整内容
  - 支持点击占位符编辑原始内容
  - 考虑显示内容预览的前几行
