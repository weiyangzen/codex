# frame_2.txt 研究文档

## 场景与职责

`frame_2.txt` 是 Codex TUI 应用服务器的 ASCII 艺术动画帧文件，属于 `dots` 动画变体系列中的第二帧。作为 36 帧循环动画的续帧，它呈现了与 frame_1 略有不同的点阵分布，共同构成流畅的动态视觉效果。

## 功能点目的

1. **动画连续性**：作为序列中的第二帧，承接 frame_1 的视觉状态，推进动画流动
2. **视觉动态**：通过点阵位置的微调变化，营造"流动"或"呼吸"的视觉效果
3. **帧序列协调**：与相邻帧（frame_1, frame_3）配合，形成平滑的过渡动画

## 具体技术实现

### 文件格式
- **尺寸**：17 行 × 40 列（与系列中所有帧保持一致）
- **字符集**：
  - `○` (U+25CB)：白色圆圈，背景/低密度区域
  - `●` (U+25CF)：黑色圆圈，主要图案元素
  - `◉` (U+25C9)：靶心圆圈，视觉焦点
  - `·` (U+00B7)：中间点，过渡/渐变效果

### 与 frame_1 的差异分析

对比 frame_1，frame_2 的主要变化：

| 区域 | frame_1 | frame_2 | 变化描述 |
|------|---------|---------|----------|
| 第 2 行顶部 | `○◉○◉○●●○○●●○` | `○◉○◉○●○○○●●○` | 第 7 个字符从 ● 变为 ○ |
| 第 3 行左侧 | `○○●◉●○●◉●○○··` | `○●●◉●◉●···○··` | 点阵密度增加 |
| 第 5 行 | `◉●○○●●●●○` | `○·◉○◉●●●●○` | 左侧图案重组 |
| 第 10 行中部 | `·○○◉◉●●●·◉●◉●·` | `○○○○○○◉◉○○·●◉ ·` | 右侧图案简化 |

这些变化创造了微妙的"点阵流动"效果，模拟某种有机运动。

## 风险、边界与改进建议

### 一致性风险

1. **尺寸不匹配**：
   - 如果 frame_2 行数/列数与其他帧不一致，会导致渲染错位
   - **检查**：所有帧应为 17 行，每行约 40 字符

2. **字符编码**：
   - 必须使用 UTF-8 编码保存
   - 避免使用组合字符或零宽字符

### 动画流畅性

当前 frame_1 到 frame_2 的过渡：
- 变化幅度适中，不会造成视觉跳跃
- 建议保持相邻帧之间变化量 < 20% 的字符差异

### 改进建议

1. **自动化验证**：
   ```bash
   # 建议添加的 CI 检查
   for frame in frames/dots/*.txt; do
       lines=$(wc -l < "$frame")
       [ "$lines" -eq 17 ] || echo "Error: $frame has $lines lines"
   done
   ```

2. **视觉一致性测试**：
   - 添加 snapshot 测试验证帧渲染结果
   - 使用 `insta` crate 进行视觉回归测试

3. **帧间差异分析工具**：
   - 开发工具自动分析相邻帧的差异
   - 检测过于突兀的帧过渡

### 维护检查清单

- [ ] 仅使用允许的字符：○ ● ◉ · 和空格
- [ ] 与相邻帧过渡平滑
- [ ] 编译无警告
