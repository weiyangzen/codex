## UI State Description
This snapshot captures the footer state after the shortcut overlay has been dismissed and the footer is showing the "Esc hint" mode. The composer is empty with the placeholder text "Ask Codex to do anything". The footer displays a single-line hint indicating that pressing Escape again will allow editing the previous message.

- `ChatComposer` - Main composer component

  - `Footer` - Bottom hint area

```rust
pub(crate) fn esc_hint_mode(current: FooterMode, is_task_running: bool) -> FooterMode {
    if is_task_running {
        current
    } else {
        FooterMode::EscHint
    }
}
```

### Esc Hint Line Rendering (`footer.rs`)
```rust
fn esc_hint_line(esc_backtrack_hint: bool) -> Line<'static> {
    let esc = key_hint::plain(KeyCode::Esc);
    if esc_backtrack_hint {
        Line::from(vec![esc.into(), " again to edit previous message".into()]).dim()
    } else {
        Line::from(vec![
            esc.into(),
            " ".into(),
            esc.into(),
            " to edit previous message".into(),
        ])
        .dim()
    }
}
```

## Behavior
1. User presses `?` to show the shortcut overlay (multi-line)
2. User presses `Esc` to dismiss the overlay
3. Footer transitions to `EscHint` mode showing the single-line hint
4. The hint indicates that pressing Escape again will trigger the backtrack/edit previous message action

## Test Context
This snapshot is generated by the test `footer_mode_overlay_then_external_esc_hint` which verifies that:
- After dismissing the shortcut overlay with Escape, the footer shows the Esc hint
- The hint uses the "esc again" format when `esc_backtrack_hint` is enabled
- The composer remains empty with the placeholder visible
