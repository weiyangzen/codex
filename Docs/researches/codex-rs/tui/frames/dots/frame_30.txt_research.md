# frame_30.txt 研究文档

## 场景与职责

`frame_30.txt` 是 Codex TUI 中 `dots` 动画系列的第30帧（索引29），在36帧动画循环中代表约83.3%的时间点。该帧展示新一轮收缩的后期状态。

## 功能点目的

- **收缩后期**：图案接近完全收缩
- **循环末期**：接近36帧循环的结束
- **准备重置**：为循环回到frame_1做准备

## 具体技术实现

### 帧特征
- 点高度集中在中心
- 边缘基本清空
- 接近收缩完成

### 技术时序
```
循环位置：83.3%
显示时间：2320ms - 2400ms
剩余帧数：6帧（约480ms）
```

### 代码集成

**帧索引**：
```rust
// frame_30.txt 对应 FRAMES_DOTS[29]
let idx = (elapsed_ms / 80) % 36;
// 当 idx == 29 时
```

- 可通过设置禁用动画

## 风险、边界与改进建议

### 技术风险
1. **循环接缝**：frame_36到frame_1的过渡需要特别平滑
2. **内存使用**：所有帧常驻内存

### 改进建议
1. **平滑过渡**：优化frame_36到frame_1的视觉过渡
2. **内存优化**：考虑按需加载或压缩存储
3. **用户控制**：允许用户调整动画速度
