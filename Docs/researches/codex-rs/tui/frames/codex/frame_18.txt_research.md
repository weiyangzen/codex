# frame_18.txt 研究文档

## 场景与职责

`frame_18.txt` 是 Codex TUI 欢迎界面 ASCII 动画序列的第 18 帧。该帧位于 36 帧动画循环的正中位置（50%），展示 Codex 标志在收缩过程中的一个关键过渡形态。

## 功能点目的

1. **循环中点**：作为第 18 帧，位于 36 帧循环的精确中点
2. **形态转折**：标志接近最小尺寸，即将开始展开
3. **视觉锚点**：在快速动画中提供一个视觉参考点

## 具体技术实现

### 文件规格
- **帧序号**：18 / 36
- **精确中点**：50%（18/36）
- **显示时间**：动画开始后约 1360ms（1.36 秒）
- **文件大小**：662 字节

### 动画循环对称性
```
帧 1-18: 展开 → 最小（前半段）
         ↑
    frame_18 (中点，最收缩)
         ↓
帧 19-36: 最小 → 展开（后半段，镜像）
```

### 技术实现
```rust
// 帧索引计算
let idx = ((elapsed_ms / tick_ms) % 36) as usize;
// 当 elapsed_ms = 1360, tick_ms = 80 时：
// idx = (1360 / 80) % 36 = 17 % 36 = 17
// FRAMES_CODEX[17] = frame_18.txt
```

## 风险、边界与改进建议

### 风险点
1. **中点同步**：作为循环中点，任何时序偏移在此处累积最明显
2. **视觉跳跃**：如果帧 17-19 之间变化过大，可能产生跳跃感

### 改进建议
1. **缓动函数**：使用缓动算法计算中间帧，而非固定帧序列
2. **自适应中点**：根据终端性能动态调整循环周期
3. **暂停功能**：允许用户暂停在特定帧（如中点）欣赏
