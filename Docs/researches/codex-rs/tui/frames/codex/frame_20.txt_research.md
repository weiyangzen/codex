# frame_20.txt 研究文档

## 场景与职责

`frame_20.txt` 是 Codex TUI 欢迎界面 ASCII 动画序列的第 20 帧。该帧展示 Codex 标志在展开过程中的一个过渡形态，位于 36 帧动画循环的后半段。

## 功能点目的

1. **展开继续**：作为第 20 帧，继续从最小状态向外展开
2. **后半段推进**：位于展开阶段的前段
3. **形态恢复**：逐步恢复到初始展开状态

## 具体技术实现

### 文件规格
- **帧序号**：20 / 36
- **循环位置**：55.6%（20/36）
- **显示时间**：动画开始后约 1520ms
- **文件大小**：662 字节

### 动画阶段
```
帧 1-12:  展开 → 收缩
帧 13-18: 收缩 → 最小
帧 19-24: 最小 → 展开（本帧位于此阶段）
帧 25-36: 展开 → 初始
```

### 技术细节
```rust
// 帧数组访问
const FRAMES_CODEX: [&str; 36] = frames_for!("codex");
// FRAMES_CODEX[19] = frame_20.txt 内容

// 当前帧计算
let idx = ((self.start.elapsed().as_millis() / 80) % 36) as usize;
// 当 idx = 19 时返回本帧
```

## 依赖与外部交互

### 内部依赖
- 依赖 frame_19 的展开起始状态
- 为 frame_21 提供过渡基础

### 外部配置
- 可通过配置禁用动画：`animations_enabled = false`
- 可通过快捷键切换变体：`Ctrl+.`

## 风险、边界与改进建议

### 边界条件
- **循环对称性**：本帧应与 frame_16 形成近似对称
- **时序精度**：80ms 间隔必须精确维持

### 改进建议
1. **对称验证**：添加测试验证前后半段的对称性
2. **动态调整**：根据终端刷新率调整帧间隔
3. **节能模式**：笔记本电池模式下降低帧率
