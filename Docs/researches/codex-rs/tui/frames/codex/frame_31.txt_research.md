# frame_31.txt 研究文档

## 场景与职责

`frame_31.txt` 是 Codex TUI 欢迎界面 ASCII 动画序列的第 31 帧。该帧展示 Codex 标志在展开过程中的一个过渡形态，位于 36 帧动画循环的后段。

## 功能点目的

1. **接近完成**：作为第 31 帧，标志非常接近初始状态
2. **循环尾声**：距离完成循环还有 5 帧
3. **准备衔接**：为无缝回到 frame_1 做准备

## 具体技术实现

### 文件规格
- **帧序号**：31 / 36
- **循环位置**：86.1%（31/36）
- **显示时间**：动画开始后约 2400ms
- **文件大小**：662 字节

### 剩余循环
```
帧 31: 86.1% (2400ms) ← 本帧
帧 32: 88.9% (2480ms)
帧 33: 91.7% (2560ms)
帧 34: 94.4% (2640ms)
帧 35: 97.2% (2720ms)
帧 36: 100%  (2800ms)
帧 1:  0%    (2880ms) ← 新一轮
```

### 技术路径
```rust
// frames.rs:35
include_str!(concat!("../frames/", $dir, "/frame_31.txt"))

// 访问
FRAMES_CODEX[30] → frame_31.txt
```

## 关键代码路径与文件引用

### 核心文件
- `frame_31.txt` - 本文件
- `frames.rs` - 宏定义
- `ascii_animation.rs` - 动画控制
- `welcome.rs` - 渲染

### 渲染调用
```
App::run()
  → Tui::draw()
    → WelcomeWidget::render_ref()
      → animation.current_frame() → FRAMES_CODEX[30]
        → frame_31.txt
```

## 依赖与外部交互

### 内部依赖
- 依赖 frame_30 的展开状态
- 为 frame_32 提供过渡基础

### 外部交互
- 用户可通过 `Ctrl+.` 切换变体
- 动画可在配置中禁用

## 风险、边界与改进建议

### 风险点
1. **循环衔接**：frame_36 到 frame_1 的过渡必须平滑
2. **时序精度**：确保 80ms 间隔精确

### 边界条件
- **时间边界**：距离循环结束还有 0.4 秒
- **对称边界**：应与 frame_5 形成对称

### 改进建议
1. **循环测试**：自动化测试验证 36 帧循环的连续性
2. **用户反馈**：收集用户对动画流畅性的反馈
3. **性能监控**：监控动画对 CPU 的影响
